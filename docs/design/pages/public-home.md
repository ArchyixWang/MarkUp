# 公开首页设计稿

## 页面定位

公开首页 `/` 是 MarkUp 的品牌入口和角色分流页。当前首页定位为专业的数据标注生产平台入口，首屏必须明确表达平台覆盖任务发布、模板搭建、数据集管理、AI 预审、人工复核和结果导出，而不是泛 AI 工具或问卷平台。

详细视觉与原型文档见：

- `public-home-style-summary.md`
- `public-home-design.md`
- `public-home-prototype.md`

## 首屏结构

- H1 使用产品型短句，当前为 `把任务发布、模板搭建和审核导出放进同一条数据生产链路`。
- 副文案说明 MarkUp 同时服务 Owner、Team Admin、Reviewer 和 Labeler，并支持文本、图片、音频、视频和多模态数据。
- 首屏右侧使用产品预览面板，展示任务发布摘要、AI / Reviewer 质量控制、任务状态、审核状态和导出就绪状态。
- 桌面端首屏底部露出下一段“平台能力”，提示用户继续浏览。

## CTA

- `开始发布任务`：未登录时打开注册弹层；已登录时进入 `/workspace?page=publish-task`。
- `浏览任务市场`：进入公开任务广场 `/tasks`。
- `进入工作台`：未登录时打开登录弹层；已登录时进入对应工作台入口。

三个 CTA 在首屏和底部收束区都保留，移动端纵向排列。

## 页面模块

1. 平台能力：任务发布与分发、模板搭建与字段映射、多模态数据管理、AI 预审与人工复核。
2. 多角色工作流：Owner、Team Admin、Reviewer、Labeler 在同一生产链路中的职责。
3. 多模态数据能力：图片、音频、视频和文本如何进入模板字段映射、AI 上下文和 Reviewer 预览。
4. AI + 人工审核：AI 输出评分、风险与建议，人工复核保留最终判断，结果导出可追踪。

## 组件与响应式

- 首页优先使用 Ant Design `Button`、`Card`、`Tag`、`Descriptions`、`Progress`、`Steps`、`Row` 和 `Col`。
- 卡片半径保持 8px，白底、浅边框和轻阴影。
- GSAP 只用于首屏进入和滚动 reveal，必须尊重 `prefers-reduced-motion`。
- 移动端从 390px 起可用，不出现页面级横向滚动，按钮单列展示，预览面板降密度堆叠。
