# 操作日志页面设计稿

## 1. 页面定位

操作日志承载关键状态流转、权限变更、审核、导出和企业管理审计日志，是 MarkUp 可追溯性的企业级入口。

页面目标：

- 查询企业内关键操作。
- 按实体、动作、操作人、时间和风险等级筛选。
- 查看单条日志详情、变更前后 diff、request_id 和关联对象。
- 支撑合规排查、质量追踪和问题复盘。

操作日志只展示审计事实，不提供直接修改业务状态的入口。

## 1.1 高频使用场景

操作日志页主要服务四类路径：

1. Team Admin 追踪成员角色、权限、邀请、移除和安全提醒等人员治理操作。
2. Owner 排查任务发布、模板发布、数据导出、积分预算治理操作和 AI 配置变更对生产链路的影响。
3. Reviewer 或平台运营复盘审核通过、打回、要求修改和批量审核等质量事件。
4. 管理员按 request_id、实体 ID、操作人和时间范围定位一次失败请求或争议操作。

首屏必须强化筛选和详情追溯能力。不要用时间线替代表格，也不要在日志详情里提供绕过业务页面的状态修改按钮。

## 1.2 审计事实边界

操作日志页只展示已经由后端写入的审计事实。前端可以把 `action`、`entity_type` 和 `changes` 转成更易读的摘要，但不能生成不存在的日志、不能补写 IP、设备、操作人或风险等级事实。

| 可由前端派生 | 不可由前端伪造 |
| --- | --- |
| 列表摘要、风险 Tag 展示、request_id 截断显示、字段 diff 排版。 | 日志时间、操作人、IP、User-Agent、审计动作、实体 ID、下载事实、权限变更事实。 |

如果后端缺少某字段，页面显示 `-`、`旧日志无记录` 或 `暂未记录`，不要用当前用户、当前时间或浏览器信息补齐历史审计。

侧栏：

- 分组：`企业管理`
- 条目：`操作日志`
- 图标：`FileTextOutlined`

面包屑：

```text
工作台 / 操作日志
```

## 3. 页面结构

```text
操作日志
├─ 固定页面标题
├─ 审计概览状态栏
├─ 搜索 / 筛选 / 导出工具条
├─ 日志表格
├─ 日志详情 Drawer
├─ 变更 diff 面板
└─ 导出日志确认弹窗
```

操作日志页面要服务排查和合规，不服务内容浏览。主视图必须保持表格、筛选和导出能力，不用时间线卡片流替代。

## 4. 页面头部

左侧：

- 标题：`操作日志`
- 副文案：`追踪状态流转、权限变更、审核、导出和企业管理的关键操作。`

右侧：

- `导出日志`
- `刷新`

导出日志仅 Owner / Team Admin 或审计权限用户可见。

## 5. 审计概览

状态栏指标：

| 指标 | 说明 |
| --- | --- |
| 今日日志 | 当前企业当天日志数量。 |
| 高风险操作 | 删除、移除成员、预算变更、provider 变更等。 |
| 权限变更 | 成员角色、权限、状态。 |
| 导出下载 | 导出创建和下载审计。 |
| 审核操作 | 通过、打回、复审、终审。 |

后端无聚合时显示当前筛选内统计。

概览只用于快速定位，不作为合规统计最终口径。需要正式报表时应使用后端聚合或导出结果。

## 6. 页面状态

### 6.1 加载态

- 日志列表加载时保留筛选条和上一次分页状态。
- 切换筛选条件后表格进入局部 loading。
- 打开详情 Drawer 时先展示基础字段，再加载 diff 和关联对象。

### 6.2 空状态

- 无日志：展示 `当前筛选条件下暂无操作日志`。
- 无权限：展示 `你没有查看企业级操作日志的权限`。
- 无 diff：详情中展示 `该操作未记录字段级变更`，不要留空。

### 6.3 错误态

- 列表加载失败：显示错误 Alert 和 `重试`。
- 详情加载失败：Drawer 保持打开，展示失败原因；若列表行已有基础字段，继续展示列表行快照，不能因为详情接口失败清空上下文。
- 导出失败：确认弹窗关闭后展示失败 toast，并保留当前筛选条件。

## 7. 筛选工具条

筛选项：

- 关键词：实体名称、操作人、request_id。
- 实体类型：team、team_member、task、template、dataset、submission、review、ai_resource、export、notification。
- 动作类型：created、updated、deleted、published、paused、resumed、finished、approved、rejected、exported、downloaded、permission_changed。
- 操作人：成员选择器。
- 风险等级：普通、重要、高风险。
- 时间范围。
- IP / 设备：后续支持。

排序：

- 最新优先。
- 最早优先。
- 高风险优先。

工具条操作：

- `重置筛选`：恢复默认最近 7 天、全部实体。
- `保存筛选`：后续可选能力，用于常用审计视图。
- `导出日志`：使用当前筛选条件，需二次确认。
- `刷新`：刷新当前分页，不改变筛选条件。

筛选默认值：

- 时间范围默认最近 7 天。
- 实体类型默认全部。
- 风险等级默认全部。
- 如果从其他页面跳转进入，优先使用跳转传入的 `entity_type`、`entity_id`、`operator_id` 或 `action`，并在筛选条上显示可清除 Tag。

深链接参数已接入工作台 URL query。进入 `/workspace?page=operation-logs` 时可携带 `keyword`、`entity_type`、`entity_id`、`operator_id`、`risk_level`、`action`、`start_date`、`end_date`；页面会用这些参数初始化筛选、展示来源 Tag，并在查询、重置、清除来源筛选时同步更新 URL。

筛选条的高级项默认折叠，避免首屏被控件占满。默认常驻项为关键词、实体类型、风险等级、时间范围和导出按钮；操作人、动作类型、IP、设备、关联对象放入高级筛选。

## 8. 日志表格

使用 Ant Design `Table`。

列：

| 列 | 内容 |
| --- | --- |
| 时间 | `created_at`。 |
| 操作 | action + 风险 Tag。 |
| 实体 | entity_type、entity_id、实体名称。 |
| 操作人 | 用户名、角色。 |
| 变更摘要 | 字段变化、状态迁移摘要。 |
| 来源 | IP、User-Agent、request_id。 |
| 关联对象 | 任务、导出、审核提交等跳转。 |
| 操作 | 查看详情。 |

高风险日志行可用左侧细色条提示，不要整行大面积红色。

表格交互：

- 点击实体名称跳转对应业务对象。
- 点击操作人打开成员详情或用户详情 Drawer。
- 点击 request_id 复制。
- 高风险日志行固定展示风险 Tag。
- 批量选择仅用于导出选中日志，不提供批量修改。

表格密度：

- 默认使用中等密度，行高控制在可扫描范围。
- 变更摘要最多两行，超出使用 Tooltip 或详情 Drawer。
- request_id 显示后 8 位，复制时复制完整值。
- IP 和 User-Agent 合并在“来源”列，User-Agent 过长必须截断。

## 10. 日志详情 Drawer

内容：

- 基础信息：时间、操作、实体、操作人、request_id。
- 风险等级。
- 变更前后：JSON diff 或字段级 diff。
- 关联业务对象。
- 请求上下文：IP、User-Agent、来源页面。
- 影响范围：任务、成员、提交、导出文件等。

敏感字段：

- API key、密码、验证码、token 不展示。
- 文件下载 URL 如含签名参数需脱敏。

详情 Drawer 分区：

1. 概览：操作、风险等级、时间、操作人、实体、request_id。
2. 变更摘要：可读摘要和状态流转。
3. Diff：字段级、JSON 或状态迁移视图。
4. 请求上下文：IP、User-Agent、来源页面、request_id。
5. 关联对象：任务、模板、数据集、成员、导出、通知等跳转入口。

Drawer 宽度建议 640-760px；复杂 JSON diff 可在 Drawer 内使用可折叠代码块，但默认只展开发生变化的顶层字段。

当前前端打开日志详情时会先用列表行快照展开 Drawer，再调用 `GET /api/v1/audit-logs/{log_id}` 拉取完整日志并刷新 request_id、diff 和请求上下文。详情加载失败时 Drawer 不关闭，保留快照并展示错误。

### 10.1 变更摘要格式

列表中的变更摘要应可读，不直接输出大段 JSON。

示例：

| 场景 | 摘要 |
| --- | --- |
| 任务发布 | `任务状态 draft -> published` |
| 成员角色变更 | `成员角色 reviewer -> owner` |
| Provider 配置 | `默认模型 gpt-4.1-mini -> gpt-4.1` |
| 导出下载 | `下载 export_20260529.csv` |
| 审核打回 | `提交被 rejected，包含审核意见` |

### 10.2 Diff 展示

Diff 面板分三种：

- 字段级 diff：普通对象变更，按字段展示 before / after。
- JSON diff：模板 schema、AI provider 配置等复杂对象。
- 状态迁移：用状态流转条展示 from / action / to。

敏感字段只展示 `已脱敏`。

Diff 呈现规则：

- 普通对象使用字段级 before / after 表格。
- 状态流转使用 `from -> action -> to` 的紧凑状态条。
- 模板 schema、Provider 配置等复杂 JSON 默认折叠，只展开发生变化的顶层字段。
- 超过 200 行的 diff 默认只显示摘要和下载/复制入口，避免 Drawer 卡顿。
- 不允许在 diff 中展示 API key、密码、验证码、token、签名下载链接完整参数。

### 10.3 风险等级

风险等级建议：

| 等级 | 事件 |
| --- | --- |
| 普通 | 查看、普通资料更新、通知已读。 |
| 重要 | 任务发布、模板发布、积分预算治理操作、审核操作。 |
| 高风险 | 删除、移除成员、权限变更、provider key 变更、导出下载。 |

若后端暂未提供 `risk_level`，前端可按 action 做临时展示，但必须在文档和代码注释中说明这是展示层推断，鉴权仍以后端为准。

风险等级展示规则：

- 风险 Tag 只用于提醒，不作为权限判断。
- 导出审计日志、Provider key 变更、批量移除成员默认高风险。
- AI 预审失败、积分预算治理异常、模板发布、任务发布默认重要。
- 普通资料更新、通知已读、查看类事件默认普通。
- 若后端返回 `risk_level`，前端不得覆盖；只在缺失时使用展示层推断。

## 11. 导出日志

导出支持：

- CSV：当前已接入后端同步下载。
- JSON：后端接口已支持。
- Excel：后续按异步导出任务补齐。

导出范围：

- 当前筛选条件。
- 时间范围。
- 实体类型。
- 风险等级。

规则：

- 导出日志本身也写入审计日志。
- 大范围导出需二次确认。
- 仅 Owner / Team Admin 或审计权限用户可导出。

导出确认弹窗：

- 展示当前筛选条件。
- 展示预计导出范围或 `后端计算中`。
- 格式选择：CSV / Excel / JSON。
- 是否包含 diff 详情。
- 是否包含 request_id。

导出后：

- 当前能力为直接下载 CSV / JSON。
- 导出日志本身写入 `audit_logs`，action 为 `audit_log_exported`。
- 大范围审计导出后续再接入异步任务、进度和下载历史。

导出交互：

- 用户点击 `导出日志` 后先展示当前筛选条件摘要。
- 时间范围超过 90 天、未选择实体类型或包含高风险日志时，确认弹窗必须明确风险。
- 导出过程中按钮进入 loading，不改变当前表格筛选和分页。
- 导出失败时保留筛选条件，并展示后端错误与 `request_id`。

## 14. 权限

- Owner / Team Admin：查看企业范围审计日志。
- Reviewer：仅查看与自己审核任务相关的审核日志。
- Agent：仅查看与自己相关的 AI 调用日志和被授权的资源治理日志。
- Labeler：默认不可访问企业级操作日志。
- Platform Admin：可查看平台级日志。

前端必须按后端权限响应控制可见范围，不用前端本地过滤代替鉴权。

## 15. 和其他页面的联动

- 企业信息页 `查看操作日志`：带 `entity_type=team`。
- 人员管理页成员详情：带 `entity_type=team_member` 或 `operator_id=user_id`。
- 任务管理页操作日志 Tab：带 `entity_type=task`、`entity_id=task_id`。
- 模板搭建页版本日志：带 `entity_type=template`。
- 资源配置页开关/provider 日志：带 `entity_type=ai_resource` 或 `resource_config`。
- 导出中心下载记录：带 `entity_type=export`。

从其他页面进入操作日志时，筛选条顶部显示来源 Tag，例如 `来自企业信息：team`、`来自成员：user_id`。用户可单独清除来源筛选，清除后恢复普通日志查询。

当前前端已支持从人员管理、企业信息和资源配置等入口传入初始筛选，并在操作日志表格上方展示来源筛选 Tag。用户点击 `清除来源筛选` 后会清空 `entity_type`、`entity_id`、`operator_id` 和 `action` 等来源条件，保留关键词、风险等级等用户手动筛选项，再重新查询日志。

当前工作台已把操作日志筛选写入 URL query。从人员管理、企业信息、资源配置等页面跳转到操作日志时，会生成可复制的 `/workspace?page=operation-logs&...` 链接；刷新页面或直接打开链接仍能恢复筛选条件。

## 16. 当前实现与后续目标

当前操作日志页面应按已有接口能力落地：

- `GET /api/v1/audit-logs?team_id={team_id}` 必须校验企业作用域。
- 列表已支持 `keyword`、`entity_type`、`entity_id`、`action`、`operator_id`、`risk_level`、`start_date`、`end_date`、分页。
- 前端默认按最近 7 天查询；时间范围使用 `DatePicker.RangePicker`，切换后会把 `start_date`、`end_date` 同步传给列表查询和当前筛选导出。`重置筛选` 恢复最近 7 天、全部实体、全部风险。
- 操作日志筛选已支持 URL query 深链接；`page=operation-logs` 与筛选字段共同决定初始页面和查询条件。
- 日志详情 Drawer 已接入 `GET /api/v1/audit-logs/{log_id}`；点击查看详情后按后端详情返回刷新 `request_id`、字段 diff 和请求上下文。
- `request_id` 已作为新写入日志字段返回；旧日志允许为空。
- 当前筛选结果可通过后端同步导出 CSV / JSON，导出动作写入 `audit_log_exported`。
- `risk_level` 当前可由后端按 action 推断返回，最终仍建议沉淀为持久化审计字段。

后续目标：

- 大范围审计日志异步导出、进度查询和下载历史。
- 关联对象名称冗余，避免列表只显示 ID。
- 风险等级写入字段而非展示推断。
- request_id 贯穿所有业务路由和历史日志回填。

## 16.1 前端实现蓝图

操作日志页是审计查询工具，首屏必须优先服务“定位一条事实”。

### 首屏布局

```text
固定标题栏
├─ 标题、副文案
├─ 导出日志 / 刷新
└─ 审计概览：今日日志、高风险、权限变更、导出下载、审核操作

主体
├─ 来源筛选 Tag（从其他页面跳转时出现）
├─ 筛选工具条：关键词、实体、风险、时间范围
├─ 高级筛选：操作人、动作、实体 ID、IP/设备预留
├─ 日志表格
└─ 日志详情 Drawer + Diff
```

### 查询体验

- 默认查询最近 7 天，避免首次加载范围过大。
- 从企业信息、人员管理、资源配置等页面进入时，顶部显示来源 Tag；用户可单独清除来源条件。
- `request_id` 支持关键词搜索和一键复制；列表中可显示后 8 位，复制完整值。
- 高级筛选默认折叠，避免首屏控件过多。
- 重置筛选恢复最近 7 天和全部实体，不清空当前页面身份。

### 详情 Drawer

Drawer 宽度建议 640-760px，分区如下：

| 分区 | 内容 |
| --- | --- |
| 概览 | 操作、实体、操作人、时间、风险、request_id。 |
| 变更摘要 | 可读摘要，不直接输出整段 JSON。 |
| Diff | 字段级 before/after、JSON diff 或状态迁移条。 |
| 请求上下文 | IP、User-Agent、来源页面；缺失显示 `暂未记录`。 |
| 关联对象 | 任务、模板、数据集、成员、导出、通知跳转入口。 |

敏感字段必须在列表、详情、复制和导出里统一脱敏。脱敏范围包括 API key、密码、验证码、token、Authorization header、签名下载链接完整参数。

### 导出日志流程

```text
点击导出日志
-> 展示当前筛选条件摘要
-> 选择 CSV / JSON / 后续 Excel
-> 大范围或包含高风险日志时二次确认
-> 下载文件
-> 后端写入 audit_log_exported
```

导出失败时保留筛选条件和分页，展示后端错误与 request_id。前端不应把当前表格数据拼成“审计导出”来绕过后端审计。

## 18. 验收标准

- 日志主视图为可筛选表格，不用时间线作为主视图。
- 能按实体、动作、操作人、风险等级和时间范围筛选。
- 单条日志详情展示 request_id、变更摘要、diff 和关联对象。
- 新写入日志应保存并返回 request_id；上线前历史日志允许显示 `-` 或 `无 request_id`。
- 高风险事件有明确视觉标识，但不使用大面积警示色。
- 敏感字段不会在列表、详情和导出中明文展示。

## 19. Hallmark 检查

- 不做时间线卡片流作为主视图；审计日志必须表格可筛选。
- 不展示敏感字段。
- 不提供绕过业务页面的状态修改按钮。
- 不伪造 IP、设备、风险等级。
- 导出审计日志本身必须可追溯。
