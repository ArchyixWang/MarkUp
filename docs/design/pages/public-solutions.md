# 公开套餐方案页设计稿

## 页面定位

`/solutions` 是 MarkUp 的公开套餐方案页，保留历史 `/publish` 兼容跳转，但不承载真实任务发布、审批或工作台操作。首页已经承担产品介绍，本页不再做完整销售叙事，主要帮助企业用户按团队阶段选择 `Free / Basic / Pro / Enterprise / More`。

工作台内的 `workspace?page=publish-task` 仍是企业端真实任务发布向导，不因本页内容调整而改变业务语义。

## 信息架构

1. 顶部轻标题区：标题为 `企业套餐方案`，说明按组织成员、活跃生产任务和数据集存储容量选择套餐；主 CTA 为 `创建企业账号`。
2. 套餐卡：页面主体优先展示五张套餐卡，分别为 `Free / Basic / Pro / Enterprise / More`；每张卡只展示适合阶段、年费、成员上限、活跃生产任务、数据集存储和 CTA。
3. 最终 CTA：保持轻量浅色收束，主按钮为 `创建企业账号`。

不再单独展示流程画布、生产工作流、FAQ、套餐对比表或额外选择指引；套餐差异由卡片承担。

## 套餐数据

套餐信息必须与组织会员 V1 口径保持一致：

| 套餐 | 年费 | 成员上限 | 活跃生产任务 | 数据集存储 |
| --- | ---: | ---: | ---: | ---: |
| Free | 0 | 3 | 3 | 3 GB |
| Basic | 999 | 10 | 5 | 20 GB |
| Pro | 3,999 | 50 | 30 | 500 GB |
| Enterprise | 19,999 | 300 | 200 | 2 TB |
| More | 联系平台定制 | 定制 | 定制 | 定制 |

More 仅作为定制入口展示，不伪造自动购买流程。

## 用户可见文案约束

- 不得出现内部定位说明、设计备注或对页面用途的自我解释。
- 不重复首页的产品介绍，不再讲完整生产线闭环。
- 文案聚焦套餐选择：适合谁、多少钱、限制是什么、下一步做什么。
- 不伪造客户 Logo、真实客户数、成交案例或不存在的社会证明。

## 视觉约束

- 页面是 pricing-first 的浅色方案页，不使用深色区块。
- 顶部轻标题区可以使用浅色品牌面板和套餐总览面板增强公开页质感，但不得重新变成完整产品销售 Hero。
- 套餐卡是第一主体，需通过色带、价格区、推荐标识和清晰权益层级强化可扫读性；Pro 高亮但不做夸张推销。
- 不再放完整对比表，避免套餐卡与表格重复。
- `Plans` 等页面结构标识使用自定义大号编号 marker，不能使用会显得小气或被布局拉伸的状态 Tag。
- 组件优先使用 Ant Design `Button`、`Card`、`Tag` 等基础组件，图标使用 `@ant-design/icons`。
- 移动端单列显示，套餐卡不得造成页面级横向滚动。
