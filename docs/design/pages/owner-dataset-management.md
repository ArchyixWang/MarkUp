# 数据集管理页面设计稿

## 1. 页面定位

数据集管理是 Owner 生产链路的入口页面，负责把外部文件导入为可检查、可修正、可预览、可导出、可映射到模板 `ShowItem` 的结构化数据集。

页面不承担模板搭建和任务发布本身，但必须为后续两步提供稳定输入：

- 模板搭建：提供可预览的字段和渲染变量。
- 发布任务：提供可映射到 `ShowItem` 的列、派生变量和数据行。
- 导出链路：提供原始数据下载和后续审核通过数据导出的入口语义。

设计风格应是生产后台工具，不做营销页式展示。信息密度可以偏高，但层级必须清晰、操作必须可追踪。

## 2. 入口与命名

### 2.1 工作台侧栏

- 当前侧栏条目 `数据集` 改名为 `数据集管理`。
- 所属分组仍为 `数据生产`。
- 图标继续使用数据库/表格语义图标，例如 Ant Design `DatabaseOutlined`。
- 点击后进入数据集管理列表页。

### 2.2 面包屑

列表页：

```text
工作台 / 数据集管理
```

修改子页面：

```text
工作台 / 数据集管理 / [数据集名字]
```

说明：

- 侧栏使用更明确的功能名 `数据集管理`。
- 面包屑中间层显示 `数据集管理`，并可点击返回数据集管理列表页。
- `[数据集名字]` 使用当前数据集名称；加载中展示 `加载中...`。

### 2.3 路由建议

如果继续使用当前单页工作台状态机，可先保持内部状态切换；如果需要深链接，建议逐步演进为：

| 页面 | 建议路径 | 说明 |
| --- | --- | --- |
| 数据集管理列表 | `/workspace/datasets` | 展示所有数据集与导入入口。 |
| 数据集修改子页面 | `/workspace/datasets/:datasetId` | 展示数据预览、字段修改、变量、导出等操作。 |

若短期不改路由，也应在 `AppShell` 面包屑中模拟同样层级，保证用户感知一致。

### 2.4 标题固定

- 工作台所有页面的页面标题区，例如 `Datasets / 数据集管理`、模板搭建、发布任务等，不应随着页面主体上下滚动离开视口。
- 在当前 shell 中，页面级 `page-heading` 应在内容区内 sticky 固定，位于全局面包屑下方，避免遮挡表格、Tabs 或弹窗。
- 固定标题只承载当前页面名称、简短说明和主操作；长列表、字段编辑和预览表格仍在标题下方自然滚动。

## 3. 用户目标

Owner 进入页面后的核心任务：

1. 查看企业已有数据集，快速判断哪个数据集可用于发布任务。
2. 导入新数据集，完成列解析和基础校验。
3. 点击某个数据集的 `修改` 进入子页面，检查数据预览。
4. 修改字段备注、展示名、映射开关和渲染变量。
5. 调整数据预览表格列宽，便于检查长文本、多模态 URL 和 JSON 字段。
6. 保存字段设置。
7. 下载/导出原始数据，或进入后续模板搭建/发布任务。

## 4. 信息架构

数据集管理拆成两个层级，不把所有功能塞在同一页。

```text
数据集管理列表页
├─ 页面标题与主操作
├─ 筛选/搜索/状态栏
├─ 数据集表格
├─ 导入数据集弹窗
└─ 删除/导出确认弹窗

数据集修改子页面
├─ 子页面标题与保存/导出/返回
├─ 数据集基础信息
├─ 数据预览
├─ 字段管理
├─ 渲染变量
├─ 多模态素材
└─ 保存状态与错误反馈
```

## 5. 列表页设计

### 5.1 页面头部

左侧：

- 标题：`数据集管理`

右侧主操作：

- `导入数据集`，Ant Design `Button type="primary"`。
- `刷新`，普通按钮或图标按钮。

不建议在列表页常驻展示大表单。导入、新增变量等操作应放在弹窗或子页面中。

### 5.2 状态栏

使用一条紧凑状态栏，不使用多个大圆角磁贴。

建议指标：

| 指标 | 来源 | 用途 |
| --- | --- | --- |
| 数据集总数 | `GET /datasets` | 当前企业数据资产规模。 |
| 总行数 | 列表聚合或前端求和 | 判断生产规模。 |
| 可映射字段数 | `columns.use_in_mapping !== false` | 判断是否已完成发布准备。 |
| 最近导入时间 | `created_at` 或 `updated_at` | 判断数据新鲜度。 |

如果后端暂无 `updated_at`，前端先不展示该指标，避免伪造。

### 5.3 搜索与筛选

列表页顶部表格上方保留轻量工具条：

- 搜索：按数据集名称、简介、字段名。
- 格式筛选：CSV、Excel、JSON、JSONL。
- 状态筛选：解析中、可用、失败、已归档。
- 快捷筛选：有多模态素材、包含派生变量、可发布映射字段不足。

当前 API 若未支持筛选参数，前端可先本地筛选；接口扩展时再下沉到后端。

### 5.4 数据集表格与卡片视图

默认使用 Ant Design `Table`，并与任务管理、模板搭建页共用生产列表表格风格：固定表头、内部滚动、右下角分页、主名称单元格、紧凑元信息列和固定操作列。筛选栏右侧提供 Ant Design `Segmented`，可切换为卡片扫描视图。卡片视图用于快速比较数据集规模、字段准备度和导出入口，不替代表格的精确列管理。

建议列：

| 列 | 内容 | 交互 |
| --- | --- | --- |
| 数据集名称 | 名称、简介一行截断 | 使用与任务/模板一致的主链接单元格视觉；点击只做当前数据集聚焦，不代替修改入口。 |
| 格式/状态 | 来源格式 Tag、解析状态 Tag | 使用 Ant Design `Tag`，不单独拉长列宽。 |
| 负责人 | 最新修改人 `updated_by_name`，无名称时回退 `updated_by`；历史数据缺少最新修改人时回退创建人 | 表格内只使用 Ant Design `Tag` 展示姓名，不再显示灰色“创建人”说明文字；用于快速判断最近维护责任人。 |
| 数据规模 | 行数、字段数、多模态素材数 | 使用两行紧凑数字栈，与任务进度列密度一致。 |
| 字段配置 | 可映射字段数、派生变量数 | 使用两行紧凑元信息，不用额外大标签组。 |
| 最近更新 | 时间 | 后端无字段时显示 `-`。 |
| 操作 | 修改、导出、删除 | 使用 Ant Design 图标按钮；`修改` 是进入子页面的主要操作，导出使用 Dropdown，删除使用 Popconfirm。 |

最近更新时间展示必须和任务管理、模板搭建共用统一时间格式化：无时区 ISO datetime 按 UTC 解析，再转换为浏览器系统时区显示；带 `Z` 或时区偏移的字符串按原时区解析。

表格行本身不绑定进入编辑动作，避免误触。进入修改必须点击操作列的修改图标。

卡片视图补充：

- 卡片使用 Ant Design `Card`、`Tag`、`Dropdown`、`Popconfirm` 和 `Pagination`。
- 卡片顶部展示格式、解析状态和更新时间；主体展示数据集名称与两行简介；负责人区展示最新修改人；中部展示行数、字段数、可映射字段；底部展示渲染变量和映射就绪状态。
- 卡片操作只保留 `修改`、`导出`、`删除`，删除必须二次确认；`表格编辑` 只存在于数据集修改子页面的表格 Tab，不在卡片底部单独暴露。三枚操作按钮必须在卡片底部容器内等分或换行排列，不允许横向溢出卡片边界。
- 卡片区内部滚动，底部固定分页器，支持每页条数和快速跳转。

### 5.5 列表页操作

| 操作 | 行为 |
| --- | --- |
| 导入数据集 | 打开导入弹窗。 |
| 修改 | 进入数据集修改子页面。 |
| 导出 | 展开格式菜单：JSON、JSONL、CSV，后续支持 Excel。 |
| 删除 | 二次确认，提示删除后对草稿任务/模板映射的影响。 |
| 刷新 | 重新请求列表，保持当前筛选条件。 |

删除前置规则：

- 如果数据集已绑定 `pending_review/published/paused/finished` 任务，应禁用删除或提示只能归档；草稿任务可在发布前检查中重新暴露缺失绑定。
- 如果当前后端无法判断绑定关系，前端先用保守文案提示风险。

## 6. 导入数据集弹窗

导入弹窗使用 Ant Design `Modal + Form + Upload`，不是页面内常驻表单。

### 6.1 交互结构

导入多模态数据时，用户通常不清楚“表格文件、媒体文件、URL 清单、模板映射”之间的关系。导入弹窗应从单表单升级为 Ant Design `Steps + Form + Upload + Table` 的向导式弹窗，默认仍保持轻量，不把高级能力暴露成一堆必填项。

```text
Step 1 选择来源
├─ 表格 / JSONL / Manifest
├─ ZIP + Manifest
└─ 外部 URL / 对象存储清单

Step 2 解析与识别
├─ 字段类型自动识别
├─ 媒体列识别
├─ 主媒体 / 上下文媒体确认
└─ 异常行提示

Step 3 预览与入库
├─ 样本预览
├─ 媒体可访问性检查
├─ AI / Reviewer 上下文预览
└─ 导入并后台处理
```

对于只有 CSV / XLSX / JSON / JSONL 的普通数据集，用户可以停留在“快速导入”路径：填写名称、选择表格文件、确认导入。系统在后台自动完成字段识别和基础媒体识别。

对于图片、音频、视频等多模态数据，导入向导必须显式展示“每行样本将绑定哪些媒体”，避免用户只上传了一批素材但没有和数据行建立关系。

### 6.2 导入来源

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| 数据集名称 | Input | 必填，2-60 字。 |
| 简介 | TextArea | 可选，建议说明来源和用途。 |
| 数据文件 | Upload | 支持 CSV、Excel `.xlsx`、JSON、JSONL；表格字段可包含媒体 URL、对象路径、文件名或附件数组。 |
| Manifest 文件 | Upload | 推荐多模态数据使用 `.jsonl`，每行一个样本，显式声明 `data/media/attachments/derived_context`。 |
| ZIP 素材包 | Upload | 可选，支持 `manifest.jsonl + media/` 的压缩包；后端解压后把相对路径解析为文件引用。 |
| 追加媒体文件 | Upload multiple | 可选，图片/音频/视频/文档文件；若未在 manifest 中绑定，则进入“未绑定素材”区。 |
| 外部素材 URL/路径 | TextArea | 可选，每行一个 URL、S3/OSS 路径或本地路径引用；导入时生成媒体资产索引。 |
| 解析设置 | Collapse 高级项 | 编码、分隔符、首行是否标题、采样行数、媒体 URL 签名策略。 |

Manifest 推荐结构：

```json
{
  "external_id": "case-001",
  "data": {
    "title": "夜间道路缺陷",
    "description": "请判断路面是否存在裂缝"
  },
  "media": [
    { "type": "image", "role": "primary", "url": "s3://bucket/images/001.jpg", "field": "image_url" },
    { "type": "audio", "role": "context", "url": "s3://bucket/audio/001.wav", "field": "voice_url" }
  ],
  "attachments": [
    { "type": "text", "name": "标注说明", "value": "只判断主车道区域" }
  ],
  "derived_context": {
    "ocr_text": "道路施工标识",
    "asr_text": "背景音无有效语音"
  }
}
```

短期兼容规则：

- 普通表格仍导入为 `rows`，不强迫用户写 manifest。
- 表格列中命中图片、音频、视频 URL 后，后端自动补齐 `row.media[]`，并保留原始列值。
- 单独上传的素材如果无法绑定到行，只进入数据集级 `media_assets`，并在页面中标记为“未绑定素材”，不会默认进入标注题目上下文。
- 多模态导入成功后，必须返回 `media_schema`、`context_schema` 和 `processing_summary`，供模板搭建和任务发布映射使用。
- 表格字段管理只展示业务字段和派生变量；行级 `media`、`attachments`、`derived_context`、`_bindings` 只作为样本视图、AI/审核上下文和 Renderer 的系统上下文，不作为普通列编辑或映射候选重复展示。

### 6.3 反馈

- 上传中：按钮进入 loading，显示解析进度或 `正在解析...`。
- 成功：关闭弹窗，列表顶部插入新数据集，toast 显示 `数据集已导入，列概览和预览已生成。`
- 失败：弹窗内显示错误原因；如果后端返回行号，展示错误行表格。
- 当前实现统一使用 Ant Design `Modal + Form + Upload`。导入成功和失败反馈使用顶部 5 秒自动消失的 Ant Design `message`，不再保留自定义常驻弹层。

## 7. 修改子页面设计

修改子页面是数据集的主工作区，点击列表行操作 `修改` 后进入。

### 7.1 子页面头部

左侧：

- 返回按钮：`返回数据集管理`
- 标题：数据集名称
- 副信息：格式、行数、字段数、状态、企业。

右侧：

- `保存修改`，主按钮。
- `导出`，下拉按钮。
- `进入模板搭建`，次级按钮，带上当前数据集作为上下文。
- `发布任务`，次级按钮，若字段映射准备不足则禁用并提示。

### 7.2 子页面布局

建议使用上方固定摘要 + 下方工作区。

```text
┌──────────────────────────────────────────────┐
│ 返回 / 标题 / 状态 / 保存 / 导出 / 后续动作       │
├──────────────────────────────────────────────┤
│ Tabs: 数据预览 | 字段管理 | 渲染变量 | 素材 | 历史 │
├──────────────────────────────────────────────┤
│ 当前 Tab 内容                                  │
└──────────────────────────────────────────────┘
```

Tab 不应切换路由；面包屑仍停留在当前数据集。Tab 导航在视觉上应相对页面左边界略向右缩进，避免贴边并与下方内容分区形成清晰层级。

### 7.3 基础信息区

基础信息不做大面积卡片，可放在 `Descriptions` 或紧凑表单中：

- 数据集名称，可编辑。
- 简介，可编辑。
- 来源格式，只读。
- 行数，只读。
- 字段数，只读。
- 状态，只读。
- 创建/更新时间，只读，若后端支持。

名称和简介保存走 `PUT /datasets/{dataset_id}`。保存成功后更新列表缓存。

## 8. 数据预览 Tab

数据预览是修改子页面的默认 Tab。

### 8.1 表格行为

使用 Ant Design `Table`，但表头需要支持拖拽调整列宽。

要求：

- 表头右侧显示 6-8px 的 resize handle。
- 鼠标按下后横向拖拽调整列宽。
- 列宽最小 96px，最大 520px。
- 双击 handle 自动适配当前列内容宽度。
- 拖拽时显示竖向参考线，不要实时大幅重排整页。
- 列宽保存在前端状态中，按 `dataset_id + column_name` 记忆；后续可接入服务端偏好。
- 移动端不提供精细拖拽，改为横向滚动和列设置抽屉。
- 当前实现预览区域最多展示前 12 列，但行数据必须来自 `GET /datasets/{dataset_id}` 返回的完整 `rows`，不能只使用列表接口中的 `preview_rows` 采样数据。表格使用 Ant Design `Table` 的固定布局、固定表头、内部滚动和底部分页，不再使用原生 `<table>`。

实现建议：

- 短期不引入新依赖，使用自定义 `ResizableHeaderCell` + Pointer Events。
- 若后续需要成熟表格能力，再评估 `react-resizable` 或虚拟表格方案。

### 8.2 预览列类型

| 数据类型 | 渲染方式 |
| --- | --- |
| text | 单元格最多两行，悬浮 Tooltip 查看全文。 |
| number | 右对齐，使用 tabular numbers。 |
| image | 小图缩略图，点击打开预览 Modal。 |
| audio | 显示文件名/URL + 播放入口，避免表格内直接铺满播放器。 |
| video | 显示文件名/URL + 预览入口。 |
| json | 折叠为 `{...}`，点击 Drawer 查看格式化 JSON。 |
| empty | 灰色 `空值` Tag。 |

多模态字段的表格单元格不能只显示 URL 文本：

- 图片：固定 72x48 缩略图 + 文件名，点击打开 `Image.PreviewGroup` 或 Drawer。
- 音频：显示音频图标、文件名、时长；点击在 Drawer 内播放，表格内不铺满播放器。
- 视频：显示封面帧、时长、处理状态；点击 Drawer 播放。
- 多媒体数组：显示媒体数量和类型 Tag，点击查看样本完整媒体列表。
- 派生上下文：OCR / ASR / caption / summary 用文本摘要展示，并标明“AI 上下文”。

### 8.3 预览工具条

- 每页条数选择：10 / 20 / 50 行，分页覆盖完整数据集行。
- 列显示设置：打开 Drawer，勾选显示列。
- 搜索当前预览：本地过滤完整行数据。
- 仅看异常：显示空值、类型不一致、URL 无法识别等行。
- 重置列宽：恢复默认宽度。

## 9. 字段管理 Tab

字段管理用于修改字段的展示与映射行为。

### 9.1 字段表格

建议列：

| 列 | 说明 |
| --- | --- |
| 参与映射 | Switch，控制是否进入发布任务 ShowItem 映射候选。 |
| 原始字段名 | 只读，保持导入源字段稳定。 |
| 展示名 | 可编辑，用于前端展示和导出重命名建议。 |
| 数据类型 | 默认只读；如允许修改，需进入确认模式。 |
| 来源 | 原始列 / 派生变量。 |
| 备注 | 可编辑。 |
| 质量提示 | 空值率、样例、类型冲突。 |
| 操作 | 查看样例、复制字段名。 |

### 9.2 字段修改边界

当前 API 稳定支持：

- `comment`
- `use_in_mapping`
- `derived_columns`

因此设计上不要直接把原始字段名做成可任意改名。推荐：

- 原始字段名 `name` 只读。
- 新增 `display_name` 或使用 `comment` 承载展示说明。
- 如果要真正重命名字段，需要后端支持数据行 key 迁移、历史映射更新和任务草稿影响检查。

### 9.3 批量操作

- 批量加入映射。
- 批量移出映射。
- 批量设置数据类型，需确认。
- 批量复制字段名。

批量操作需要显示影响数量，并支持撤销或保存前取消。

## 10. 渲染变量 Tab

渲染变量是把原始列加工成更适合模板展示和发布映射的派生列。

### 10.1 变量列表

使用 `Table`：

| 列 | 内容 |
| --- | --- |
| 变量名 | 派生列名。 |
| 类型 | text / number / image / audio / video / empty。 |
| 来源列 | 绑定的原始列，可为空。 |
| 表达式 | `{value}` 或 `合同：{title}`。 |
| 映射 | 是否加入发布映射候选。 |
| 首行预览 | 展示表达式结果。 |
| 操作 | 修改、删除、复制。 |

### 10.2 新增/修改变量

使用 Ant Design `Modal` 或 `Drawer`：

- 变量名。
- 数据类型。
- 来源列。
- 默认值。
- 表达式。
- 备注。
- 是否加入发布映射候选。
- 首行实时预览。

表达式规则：

- 支持 `{value}` 表示来源列值。
- 支持 `{列名}` 引用同一行其他字段。
- 不执行任意 JavaScript。
- 错误占位符在保存前给出字段级提示。

### 10.3 删除变量

删除派生变量前检查：

- 是否被当前模板预览使用。
- 是否被草稿任务 `column_mapping` 使用。

如果后端暂不支持引用检查，前端展示保守确认文案。

## 11. 多模态素材 Tab

多模态素材用于管理随数据集上传的图片、音频、视频文件或外部 URL。

展示内容：

- 素材文件名/URL。
- 类型：image / audio / video / file。
- 关联字段名。
- 关联样本数。
- 角色：主媒体 / 上下文媒体 / 证据附件 / 未绑定。
- 处理状态：待处理 / 可用 / 转码失败 / 访问失败。
- 派生结果：缩略图、波形、关键帧、OCR、ASR、caption、summary。
- 预览入口。
- 复制 URL/路径。

当前设计不要求实现框选、涂抹或音视频片段标注组件，这些属于后续专用标注控件。

### 11.1 样本视图 Tab

多模态数据集详情页新增 `样本视图` Tab，帮助用户以“标注员将看到什么”的方式检查数据，而不是只看表格。

布局：

```text
┌──────────────┬──────────────────────────────┐
│ 样本列表       │ 样本预览                       │
│ 搜索/异常筛选   │ 主媒体 / 上下文 / 原始字段 / 派生上下文 │
└──────────────┴──────────────────────────────┘
```

交互：

- 左侧列表显示 `external_id / row_index / 模态 Tag / 异常状态`。
- 右侧上方展示主媒体：图片、音频、视频根据类型使用专用预览组件。
- 右侧下方使用 Tabs 展示 `原始字段`、`媒体列表`、`派生上下文`、`附件`。
- 右侧固定保留“复制样本 JSON”和“查看 AI/审核上下文”入口。
- 样本视图不修改数据，只用于检查导入结果和下游上下文。

### 11.2 AI / 审核上下文 Tab

新增 `AI / 审核上下文` Tab，用于提前预览当前数据集行进入大模型助手、AI 预审和 Reviewer 页面时的上下文。

展示内容：

- 当前样本的 `QuestionContext` JSON 摘要。
- 发送给多模态模型的媒体内容块：图片 URL / 音频 URL / 视频关键帧。
- 发送给纯文本模型的降级上下文：OCR、ASR、caption、summary、字段摘要。
- Reviewer 可见上下文：原始媒体、标注答案、AI 预审依据、引用片段。
- 被排除的字段：不进入 AI 上下文或 Reviewer 上下文的字段要明确标记。

这个 Tab 的目标不是让用户配置模型，而是让数据集 Owner 在导入阶段就能检查“AI 和 Reviewer 到底会看到什么”。

## 12. 模板搭建与数据映射兼容

多模态能力必须兼容模板搭建时的 schema 导入、参考数据集绑定和任务发布时的数据映射。核心规则是：旧模板继续使用 `ShowItem.config.content_field`，新模板可以声明更完整的 `ShowItem.config.binding`。

推荐 binding 结构：

```json
{
  "source_type": "media",
  "media_type": "image",
  "role": "primary",
  "field": "image_url"
}
```

支持的数据来源：

| source_type | 用途 |
| --- | --- |
| `column` | 普通数据列，兼容旧 `content_field`。 |
| `media` | 行级媒体，如主图片、上下文音频、证据视频。 |
| `derived_context` | OCR、ASR、视频关键帧、caption、summary。 |
| `attachment` | 说明文档、补充文本、证据附件。 |

模板搭建页规则：

- 选择参考数据集后，右侧属性面板的 ShowItem 绑定区展示“数据源选择器”，不是只展示列名。
- 数据源选择器按 `普通列 / 媒体 / 派生上下文 / 附件` 分组。
- 导入 schema 时，如果 ShowItem 带 `binding`，系统基于参考数据集校验数据源是否存在。
- 如果旧 schema 只有 `content_field`，系统自动转成 `{ source_type: 'column', column_name: content_field }` 的兼容展示。
- 模板 AI 生成 ShowItem 时，应优先读取参考数据集的 `media_schema` 和 `context_schema`，为图片、音频、视频展示项生成 binding。

任务发布映射规则：

- 当前 `column_mapping` 保留，用于旧 API 和旧任务草稿。
- 新增 `mapping_config`，用于保存完整多模态数据源映射。
- 发布任务时优先使用 `mapping_config` 生成 `question.content.media` 和 `question.content.derived_context`；没有 `mapping_config` 时回退到 `column_mapping`。

兼容保存示例：

```json
{
  "column_mapping": {
    "show_image": "image_url"
  },
  "mapping_config": {
    "show_image": {
      "source_type": "media",
      "media_type": "image",
      "role": "primary",
      "field": "image_url"
    }
  }
}
```

## 13. 导出设计

### 13.1 原始数据下载

列表页和子页面均提供原始数据下载：

- JSON
- JSONL
- CSV
- Excel，若后端未支持则先置灰并提示 `后端待支持`

当前 API 为：

```text
GET /api/v1/datasets/{dataset_id}/download?format=json|jsonl|csv
```

由于列表页和详情页都位于固定标题与滚动容器内，下载 Dropdown 统一挂载到 `document.body`，避免被固定面板裁切或遮挡。

### 13.2 审核后数据导出

审核通过后的标注结果导出不应混在数据集原始下载里，应跳转或引导到导出中心：

- 创建导出任务。
- 配置字段 include/exclude/rename。
- 选择是否包含审核记录。
- 查看导出进度和历史。

数据集页面只提供入口，不承载完整导出中心。

## 13. 状态设计

## 14. 自动保存

- 数据集详情页中的名称、简介、字段备注和 `use_in_mapping` 修改，与任务管理页面一致，采用前端 5 秒空闲自动保存。
- 自动保存成功后显示 `数据集已自动保存。`，5 秒后自动消失；手动点击 `保存修改` 后仍显示明确保存成功反馈。
- 自动保存只复用现有 `PUT /datasets/{dataset_id}` 能力；保存成功后刷新数据集最新修改人，不新增自动保存专用字段。

### 13.1 加载态

- 列表页：表格 skeleton。
- 子页面：标题骨架 + 表格 skeleton。
- 预览表格：列头 skeleton，不显示空白大块。

### 13.2 空态

没有数据集：

- 标题：`还没有数据集`
- 说明：`导入 CSV、Excel、JSON 或 JSONL 后，系统会自动解析字段并生成预览。`
- 操作：`导入数据集`

没有预览行：

- 标题：`暂无可预览数据`
- 说明：`文件可能为空，或解析失败。请检查导入文件。`

没有可映射字段：

- 提示：`当前没有字段加入发布映射候选，请在字段管理中开启至少一个字段。`

### 13.3 错误态

错误信息必须靠近发生位置：

- 导入错误显示在导入弹窗内。
- 保存字段错误显示在字段管理工具条下方。
- 下载错误用 toast + 可重试按钮。
- 表格解析错误如包含行号，展示错误行号和原因。

### 13.4 未保存状态

字段管理和变量修改需要未保存提示：

- 页面标题右侧显示 `有未保存修改` Tag。
- 离开子页面前弹出确认。
- 保存成功后显示 `已保存`，并刷新数据集详情。

## 14. 权限与边界

| 角色 | 权限 |
| --- | --- |
| Team Admin / Owner | 可导入、修改、删除、下载数据集。 |
| Reviewer | 默认不可修改，可根据企业权限只读查看。 |
| Labeler | 不进入数据集管理。 |
| Agent | 默认不进入数据集管理，除非企业授权。 |

所有接口都必须携带：

- `Authorization: Bearer <access_token>`
- `X-Team-ID: <team_id>`

前端不能只靠隐藏按钮控制权限，后端仍需校验。

## 16. 前端实现注意事项

1. 先把侧栏文案和 breadcrumb 改正确，再拆列表页和子页面。
2. 列表页只展示数据集列表、筛选和导入；字段修改放入子页面。
3. `修改` 进入子页面，不用行点击直接跳转。
4. 数据预览列宽拖拽必须只影响表格，不造成整个页面横向溢出。
5. 表格横向滚动只能出现在预览表格容器内，页面主体不能出现横向滚动。
6. 列宽拖拽要有键盘替代：列设置 Drawer 内提供默认宽度/紧凑/宽松选项。
7. 修改字段时区分原始字段名和展示名，不允许无提示地改动数据行 key。
8. 保存按钮应只在有未保存修改时高亮。
9. 导入、保存、删除、下载都需要 loading 和错误反馈。
10. 不要用多个大磁贴堆页面，优先使用表格、工具条、紧凑摘要和工作区布局。

## 17. API 依赖与缺口

当前已有：

- `GET /api/v1/datasets`
- `POST /api/v1/datasets`
- `GET /api/v1/datasets/{dataset_id}`
- `PUT /api/v1/datasets/{dataset_id}`
- `GET /api/v1/datasets/{dataset_id}/download?format=json|jsonl|csv`
- `DELETE /api/v1/datasets/{dataset_id}`

建议后续补充或确认：

| 需求 | 当前处理 |
| --- | --- |
| 数据集真实重命名字段 | 需要后端支持行 key 迁移和引用检查。 |
| `display_name` 字段展示名 | 建议新增，避免改原始字段名。 |
| 列宽偏好服务端保存 | 可先用 localStorage，后续加用户偏好接口。 |
| 数据集绑定任务检查 | 删除前需要后端返回是否被任务引用。 |
| Excel 原始下载 | 当前下载 API 未列出 Excel，后续补齐。 |
| 解析错误行号 | 导入失败应返回 row、column、error。 |

## 19. Hallmark 检查

- 结构：列表页与修改子页面分离，避免把所有功能塞进一个密集页面。
- 层级：主任务是管理数据集，字段与预览进入二级工作区。
- 克制：不用营销式大卡片，不堆装饰性图形。
- 专业性：用表格、工具条、抽屉、弹窗和状态标签表达生产后台语义。
- 可用性：修改入口明确，危险操作有确认，未保存状态可感知。
