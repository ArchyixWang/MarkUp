# 平台运营工作台设计稿

## 页面定位

平台运营工作台是 MarkUp 平台方的经营后台，路由为 `/platform`，独立于企业 `/workspace`。平台管理员不默认进入企业工作台。

## 页面结构

```text
平台工作台
├─ 经营总览
├─ 结算流水
├─ 认证审核
├─ AI Provider
└─ 平台规则
```

## 设计约束

- 使用 Ant Design `Statistic`、`Table`、`Tabs`、`Drawer`、`Modal`、`Form`、`Tag`、`Alert` 和 `Empty`，经营趋势使用 `@ant-design/charts` 的折线或面积图。
- 页面保持数据密集型后台风格，不使用营销页 Hero、装饰性卡片堆叠或大圆角组件。
- 财务金额在平台工作台默认只展示 `积分`，不在表格、详情和提示里反复展示 `1 积分 = 1 元`；固定关系仅保留在文档和 API 兼容字段中。
- 经营总览展示近 30 天服务费趋势，趋势数据来自 `/platform/workbench.settlement_trend`，不使用前端 mock；趋势必须是折线/面积图等经营图表，不使用 30 条 `Progress` 明细条。
- 提现按当前产品口径自动通过：企业提现与标注员提现都不进入平台人工审核队列，平台工作台不提供“提现处理”一级页面或审核弹窗。
- 结算、企业认证、资质认证列表必须提供受控筛选和分页；详情必须通过 Drawer 展示材料或上下文后再审核。
- 认证审核页使用企业认证与标注员资质双 Tab。
- `/platform` 所有一级页复用工作台固定首栏：54px 高度、一行标题与右侧状态标签，不展示多行说明造成跳动。
- 平台一级页切换由 URL 查询参数驱动，不在导航点击时先行改本地页状态，避免旧 URL effect 造成页面回弹或闪烁。
- 企业认证与标注员资质详情 Drawer 必须展示结构化材料文件行，支持 `url`、`file_id`、文件名、MIME、大小等字段；只有 `file_id` 时前端按材料来源生成受保护预览地址。审核提交成功后详情 Drawer 立即关闭并回到表格。
- 经营总览复用企业工作台 dashboard 结构与类名语义：`dashboard-scroll-area`、`dashboard-cockpit`、`dashboard-kpi-strip`、`dashboard-chart-grid`、`dashboard-main-grid` 和 `dashboard-panel`，只保留平台数据桥接样式，避免另起一套平台视觉系统。
- 结算流水、认证审核、AI Provider 和平台规则页同样对齐经营总览的白底工作区节奏：页面根容器、Tabs、筛选栏、表格外壳、Provider 双栏和规则表单都使用白底、`8px` 主间距、`6-8px` 圆角和浅边框，不出现大面积灰底或松散卡片墙。

## 经营总览密铺规则

- 经营总览首屏必须按运营后台密铺处理：12 栏网格、统一 `8px` gap，不再用松散纵向卡片堆叠。
- KPI 区是单条横向指标带，高度稳定在约 `72px`；标题使用小字号，主数值紧凑展示，指标间用细分割线区分。
- 中部区域固定为趋势图 `8` 栏、审核待办 `4` 栏；底部区域固定为最近结算 `8` 栏、运营摘要 `4` 栏。
- 趋势、待办、最近结算、运营摘要都需要稳定高度；最近结算表格在面板内部滚动，避免记录数量变化导致模块位置跳动。
- 移动端按单列顺序堆叠，但仍保持 `8px` gap、紧凑 padding 和同一信息顺序。

## 2026-05-31 Addendum

- 平台工作台导航新增 `AI Provider` 一级页面，路径为 `/platform?page=providers`。
- `AI Provider` 页面采用“左侧配置列表 + 右侧详情 + 宽 Drawer 新增/编辑”的高密度企业后台形态，只管理 `scope=platform` 的共享路由。
- `AI Provider` 新增/编辑 Drawer 内必须提供“测试连接”按钮；创建态优先走草稿测试接口，确认连通后再保存。
