# MarkUp 首页原型说明

## 1. 文字版原型结构

1. 顶部品牌区
2. 首屏 Hero
   - 标题
   - 简述
   - 角色标签
   - 三个 CTA
   - 右侧产品预览
3. 平台能力概览
4. 多角色工作流
5. 多模态数据能力
6. AI + 人工审核闭环
7. 底部收束 CTA
8. 公共页脚

## 2. 布局说明

### 顶部品牌区

- 轻量品牌露出，不占高。
- 不做传统营销站的超大导航栏。
- 如有登录态，仅展示必要入口。

### 首屏 Hero

- 桌面端左右分栏。
- 左侧是标题、说明、标签和 CTA。
- 右侧是产品型卡片组合：任务发布摘要、模板数据映射、AI 预审、人工复核、导出状态。
- 1280 宽度下仍保持可读，不压扁。

### 平台能力概览

- 采用 4 张能力卡或状态条。
- 每张卡只表达一件事：能做什么、对应什么页面。

### 多角色工作流

- 用步骤条或角色流展示 Owner、Team Admin、Reviewer、Labeler 的位置。
- 不做抽象角色海报。

### 多模态数据能力

- 用 2 到 3 个能力块说明图片、音频、视频和文本如何进入平台。
- 搭配一块数据预览风格的面板。

### AI + 人工审核闭环

- 用一条清晰的流程卡说明：生成 -> 预审 -> 复核 -> 导出。
- 重点突出人工确认和审计留痕。

### 底部收束 CTA

- 重复最重要的动作入口。
- 桌面端保持紧凑，移动端纵向堆叠。

## 3. 动效说明

- Hero 先出现标题，再出现副文案、标签和 CTA，最后显示右侧预览。
- 能力卡和工作流在滚动进入视口时轻量浮入。
- 预览面板中的状态条或高亮仅做轻微变化，不做复杂动画。
- reduced motion 下禁用位移和延时，只保留静态展示。

- `HomeMultimodalPanel`

- `HomeCTASection`

## 5. Ant Design 组件使用说明

### 使用 Ant Design 的部分

- `Button`
- `Card`
- `Tag`
- `Statistic`
- `Steps`
- `Progress`
- `Divider`
- `Space`
- `Row` / `Col`（可选）

### 使用自定义布局的部分

- Hero 双栏总布局
- 预览面板内部的产品拼接
- 多模态示意中的流程连线
- GSAP 动效的进入顺序与轻量高亮

## 6. 设计边界

- 不使用大面积渐变背景。
- 不做高饱和抽象装饰。
- 不引入问卷类、消费类或泛 AI SaaS 的营销视觉。
- 文字必须围绕数据标注、模板、数据集、审核和导出。
