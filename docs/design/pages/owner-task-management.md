# 任务管理页面设计稿

## 1. 页面定位

任务管理是 Owner / Team Admin 管理企业标注任务的主入口，承载任务列表、任务状态控制、发布前准备检查、新建任务、修改任务、导出和删除等生产管理动作。

页面不应把任务列表、创建表单、题目预览、审核配置全部压在一个首屏里。任务列表负责快速扫描和批量操作；新建和修改进入独立子页面，承载完整配置和发布校验。

设计基线：

- 使用 Ant Design `Table`、`Form`、`Tabs`、`Modal`、`Drawer`、`Dropdown` 和 `Tag`。
- 保持企业后台工具风格，信息密度高但层级明确。
- 状态机、权限和发布约束必须在界面上可见，不能只依赖提交失败提示。
- 任务管理页面及新建任务子页面统一使用浅蓝色作为模块主色。
- 任务管理关注“任务生命周期”，发布任务页面只是任务子页面中的创建/编辑流程。

## 2. 入口与命名

### 2.1 工作台侧栏

- 侧栏条目：`任务管理`。
- 所属分组：`数据生产`。
- 图标：继续使用火箭/发布语义图标，例如 Ant Design `RocketOutlined`。
- 点击后进入任务管理列表页，而不是直接进入发布任务表单。
- `数据生产` 分组内的侧栏顺序为 `数据集管理 -> 模板搭建 -> 任务管理`，任务管理位于生产链路的最后一步。

### 2.2 面包屑

列表页：

```text
工作台 / 任务管理
```

新建任务子页面：

```text
工作台 / 任务管理 / 新建任务
```

修改任务子页面：

```text
工作台 / 任务管理 / [任务名称]
```

说明：

- 面包屑中的 `任务管理` 可点击返回任务列表。
- `[任务名称]` 加载中显示 `加载中...`。
- 若新建/修改页面存在未保存变更，点击面包屑返回时必须二次确认。
- 独立新建任务页面也必须走同一条面包屑链路，即 `工作台 / 任务管理 / 新建任务`，不要退化成仅显示 `发布任务`。

### 2.3 路由建议

短期可继续使用工作台内部状态机；后续为支持深链接，建议演进为：

| 页面 | 建议路径 | 说明 |
| --- | --- | --- |
| 任务管理列表 | `/workspace/tasks` | 展示当前企业全部任务。 |
| 新建任务 | `/workspace/tasks/new` | 创建草稿并进入配置流程。 |
| 修改任务 | `/workspace/tasks/:taskId` | 查看、编辑、发布和状态控制。 |

## 3. 用户目标

Owner 进入任务管理后的核心目标：

1. 查看当前企业全部任务，判断任务状态、进度、质量和风险。
2. 通过状态、关键词、Owner、Reviewer、标签、分类、难度、截止时间快速筛选任务。
3. 新建任务草稿，完成基础信息、模板与数据、奖励、分发、审核、AI 和资质配置。
4. 修改草稿任务的完整配置；修改已发布任务时只允许编辑说明、富文本和标签。
5. 发布前检查模板、数据集、题目、列映射、审核员、AI、资质和截止配额是否满足要求。
6. 控制任务状态：发布、暂停、恢复、结束。
7. 导出任务数据，查看异步导出进度和下载历史。
8. 删除仅限草稿任务；已发布或有提交的任务只能结束、归档或导出。


## 5. 列表页设计

### 5.1 页面头部

左侧：

- 标题：`任务管理`
- 副文案：`管理任务草稿、审核发布、收集状态、质量进度和数据导出。`

右侧：

- 主按钮：`新建任务`，进入新建任务子页面。
- 次级按钮：`刷新`。
- 可选下拉：`导出任务清单`，导出列表元数据，不等同于导出标注结果。

页面标题区必须固定在面包屑下方，不随表格滚动离开视口。

当前已落地：标题区提供 `导出任务清单` 下拉按钮，支持导出当前筛选条件下的 CSV 或 JSON 任务元数据。该导出只覆盖列表字段和统计摘要，用于项目盘点、PM 周报和归档，不包含提交答案、审核记录或正式标注结果；正式结果仍走 `统计与导出` Tab 的异步导出任务。

修改任务详情页复用固定标题栏和任务准备度概览条，二者必须上下紧贴；标题栏和任务准备度概览条之间不得露出灰色页面背景、边框缝隙或额外 gap。任务准备度概览条应采用与模板搭建、数据集管理等生产页一致的连续状态条样式：白底、无内部卡片 gap、仅用中性分隔线分栏，状态值用语义色强调，不使用独立圆角卡片组。状态概览、消息提示和 Tab 主体不得因为标题栏高度或页面 gap 形成明显空白。

### 5.2 状态概览

使用紧凑状态栏，不使用大面积营销卡片。

建议指标：

| 指标 | 来源 | 用途 |
| --- | --- | --- |
| 全部任务 | `GET /tasks` 聚合 | 当前企业任务规模。 |
| 自动保存 | `status=draft + auto_saved=true` | 本质仍是草稿，列表置顶显示自动保存版本。 |
| 草稿 | `status=draft + auto_saved=false` | 待补配置或待发布。 |
| 待审核 | `status=pending_review` | Owner 提交发布后等待 Team Admin 审核。 |
| 收集中 | `status=published` | 当前可被领取或分发。 |
| 已暂停 | `status=paused` | 需处理异常或等待恢复。 |
| 已结束 | `status=finished` | 可导出或复盘。 |
| 即将截止 | `deadline` | 提前发现交付风险。 |

状态概览数字必须基于当前搜索、分类、难度等非状态筛选后的任务全集保持固定；点击 `草稿`、`待审核`、`收集中`、`已暂停`、`已结束` 只改变下方列表展示，不反向改变概览条里的各状态数字。

### 5.3 搜索与筛选

筛选条位于表格上方，保持单行优先；较窄屏幕折叠到 `筛选` 抽屉。

字段：

- 关键词：任务标题、描述、标签。
- 状态：自动保存、草稿、待审核、收集中、已暂停、已结束。
- Owner：任务负责人。
- Reviewer：审核员。
- 标签：多选。
- 分类：文本 / 图片 / 音频 / 视频；新建任务基础信息使用多选，多个模态不再通过单独的“多模态”按钮选择。
- 难度：简单 / 中等 / 困难。
- 截止时间：日期范围。
- 发布准备：模板缺失、数据集缺失、列映射缺失、AI 预审配置缺失、人工复审配置缺失。

当前 `GET /tasks` 已约定支持状态、关键词、owner、reviewer、标签筛选；分类、难度、截止时间和准备度可先前端筛选，后续下沉到接口。

### 5.4 任务表格与卡片视图

默认使用 Ant Design `Table` 作为主要列表；筛选栏右侧提供 Ant Design `Segmented`，可切换到卡片扫描视图。卡片视图参考任务广场卡片的信息企业，但保持企业端生产管理密度：状态、分类、难度、进度、奖励、审核员、标签和行级操作都必须可见。

建议列：

| 列 | 内容 | 交互 |
| --- | --- | --- |
| 任务名称 | 标题、分类、难度、标签、简介一行截断 | 标题可点击进入修改子页面。 |
| 状态 | `draft + auto_saved / draft / pending_review / published / paused / finished` | 灰色自动保存、灰色草稿、橙色待审核、绿色收集中、红色已暂停、蓝色已结束。 |
| 负责人 | 任务发布人 `owner_name`，无名称时回退短 `owner_id` | 使用 Ant Design `Tag` 展示；列标题统一为负责人，单元格标签显示发布人。 |
| 数据与模板 | 模板名、数据集名、题目数、映射完整度 | 缺失项展示 Warning Tag。 |
| 数据进度 | 总数据条数、待人工审核、已入库、打回 | 表格使用紧凑数字组；卡片使用入库进度条和四项指标。`submitted` 按当前待人工审核题目解释，`approved` 按已入库解释，`rejected` 按已打回解释。 |
| 质量 | 通过率、AI 异常数、人工待审数 | 无数据时显示 `-`。 |
| 审核/AI | Reviewer 摘要、AI 预审开关 | 悬停或详情展示审核员列表。 |
| 截止与奖励 | 截止时间、奖励模式和积分 | 即将截止高亮。 |
| 最近更新 | `updated_at` 或创建时间 | 支持排序。 |
| 操作 | 修改、复制、发布/暂停/恢复/结束、导出、删除 | 更多操作收进 `Dropdown`。 |

最近更新时间展示必须和数据集管理、模板搭建共用统一时间格式化：无时区 ISO datetime 按 UTC 解析，再转换为浏览器系统时区显示；带 `Z` 或时区偏移的字符串按原时区解析，避免任务刚保存后列表时间与系统时间不一致。

行点击只做选中或打开侧边摘要；进入编辑必须点击标题或 `修改`，避免误触。

行内 `更多` Dropdown 必须挂载到 `document.body` 或等效顶层容器，并使用高于表格滚动区的弹层层级，避免候选菜单被下方任务行遮挡。

任务结果查看与导出入口放在列表页行内 `更多` Dropdown 中。非草稿任务点击 `查看结果 / 导出` 打开 Ant Design `Drawer`，Drawer 内展示任务结果统计、导出配置、字段映射和下载历史；草稿任务该入口置灰并提示发布后可用。导出配置至少支持 JSON / JSONL / CSV / Excel 四种格式，支持按题目状态和日期范围筛选，支持是否包含审核记录。字段映射区允许勾选导出字段、重命名列名，并支持 `content.*`、`answers.*` 这类动态字段通配。下载历史表展示异步导出任务状态、进度、文件大小、下载次数、创建时间和下载/取消操作；即使当前后端先同步生成文件，前端也必须保持异步任务和历史列表模型。

表格多选后仍必须保持 Ant Design 固定表头完整可见：批量操作栏占用独立固定高度，表格 body 高度需扣减批量栏高度；表头区不可被 flex 布局压缩，也不能被下方数据渲染层覆盖。

卡片视图补充：

- 卡片使用 Ant Design `Card`、`Tag`、`Progress`、`Checkbox`、`Dropdown` 和 `Pagination`。
- 任务卡片顶部展示状态、分类、难度和批量选择 `Checkbox`；主体展示任务名、两行描述、奖励和分发策略；负责人区展示发布人；中部展示已入库 / 总数据的入库进度；指标栏展示总数据、待人工审核、已入库、打回和审核员数；底部展示标签与 `修改` / `更多` 操作。
- 卡片底部 `更多` Dropdown 同样挂载到顶层容器，弹层不能被相邻卡片遮挡。
- 卡片网格只在列表主体内部滚动，底部固定分页器，支持每页条数和快速跳转。
- 卡片多选必须复用表格同一批量操作栏，避免表格和卡片两套批量逻辑。

### 5.5 行操作规则

| 状态 | 允许操作 | 禁止/限制 |
| --- | --- | --- |
| 草稿 | 修改、发布、复制、删除 | 发布前必须完成模板和题目。 |
| 发布中 | 查看详情、暂停发放、结束、复制、导出 | 不允许直接修改；详情页允许只读查看配置、题目、审计和导出记录；必须先暂停发放，再修改说明、富文本和标签。 |
| 已暂停 | 修改说明/富文本/标签、恢复、结束、复制、导出 | 不允许领取新题，不能修改模板、数据集、配额、审核员等敏感配置。 |
| 已结束 | 复制、导出、查看统计、查看日志 | 不允许恢复为发布中。 |

`删除` 只对草稿任务启用。非草稿任务的危险操作统一使用 `结束任务` 或后续 `归档`。

负责人转交属于行级管理动作，放入 `更多` 菜单并使用短表单 Modal。Modal 展示当前任务、当前负责人、目标负责人候选列表和转交原因；目标负责人使用 Ant Design `Select showSearch`，候选只包含当前企业 active 的 Team Admin 或 Owner，不允许自由输入任意用户 ID。目标负责人未选择时 Select 也必须占满弹窗表单整行，不能按 placeholder 或空值内容收缩。提交后调用 `POST /tasks/{task_id}/owner-transfer`；成功后刷新列表并在任务操作日志中出现 `task_owner_transferred`。弹窗内不再额外展示说明型警告文案，避免重复解释已由权限和接口约束保证的转交边界。

### 5.6 批量操作

当用户勾选表格行后，在筛选条下方显示批量操作栏：

- 批量导出：创建异步导出任务，支持格式和字段配置。
- 批量结束：仅允许发布中或暂停任务，需二次确认。
- 批量打标签：仅允许草稿或已暂停任务；收集中任务必须先暂停发放后才能修改标签。

批量操作必须显示可操作数量和被跳过原因，避免静默失败。

当前已落地批量结束、批量导出和批量打标签：列表表格支持多选，批量操作条会展示已选数量、可结束数量和跳过数量；仅 `published`、`paused` 任务会逐项调用状态接口结束，草稿和已结束任务会跳过并在成功提示中说明。批量导出会为 `published`、`paused`、`finished` 任务逐项创建导出任务，支持格式、数据状态和是否包含审核记录配置；草稿任务跳过。批量打标签采用“追加标签并去重”策略，逐项调用任务更新接口；后端只允许草稿或已暂停任务更新，收集中任务必须先暂停发放。

暂停和结束都必须在执行前检查题目状态：

- `暂停发放` 只停止未领取数据继续发放，不回收已领取数据，也不影响已通过数据的积分结算；`claimed`、`submitted`、`rejected` 数据继续按原链路提交、审核和处理，不阻止暂停。
- `结束任务` 会停止全部未领取数据继续发放，且结束后不可恢复；如果存在 `claimed`、`submitted` 或 `rejected`，则阻止结束并提示先完成提交、审核和打回处理。

## 6. 新建/修改子页面

### 6.1 子页面头部

左侧：

- 返回按钮：`返回任务管理`。
- 标题：新建时显示 `新建任务`；修改时显示任务名称。
- 副信息：状态、创建者、企业、最近保存时间。

右侧：

- 标题栏不展示常驻 `保存草稿`。填写有效内容后自动保存；底部可以保留低权重 `手动保存`。
- `发布任务`：仅最后一步展示；打开发布前检查弹窗。
- `暂停` / `恢复` / `结束`：按状态展示。
- `导出`：对发布中、暂停、结束任务展示。
- `更多`：复制任务、查看操作日志、删除草稿。

有未保存变更时显示 `有未保存修改` Tag，并在离开页面前确认。

### 6.2 水平步骤条

子页面标题下方展示 Ant Design `Steps` 横条，不再使用左侧竖向 checklist，也不再把所有配置段一次性压成同屏长表单。步骤条从左到右依次为：

```text
基础信息 -> 模板与数据 -> 分发与奖励 -> AI 预审 -> 人工复审 -> 用户协议 -> 确认发布
```

状态：

- 已完成：绿色。
- 当前步骤：浅蓝色。
- 未到达：灰色。

点击任一步骤可跳转到对应工作区。当前步骤标题会固定显示在左侧工作区顶部，右侧摘要同步展示已完成配置。模板选择只展示已发布模板版本，不把草稿模板混入发布链路。

说明：

- 新建任务工作区采用双栏布局，左侧只显示当前步骤的完整配置，右侧只显示发布摘要。
- 步骤切换后，右侧摘要必须立即保留已完成步骤信息，例如已选模板、已选数据集、映射完成度、分发资质、奖励方式、AI 预审、人工复审和用户协议配置。
- 发布摘要应使用紧凑文案解释数字含义：积分为 0 或未填时显示 `未填写`，百分比字段必须带业务含义与 `%`，例如 `最低准确率 80%`、`通过阈值 90%`，不能只展示裸数字或裸百分比。
- 发布摘要里的标题、模板名、数据集名、协议名等长文本必须自动换行，不使用省略号截断核心信息；摘要面板自身滚动承载超长内容。
- 页面整体不依赖浏览器滚动承载流程，左右两栏各自内部滚动，标题栏和步骤条固定在视口内。

### 6.3 基础信息 Tab

基础信息页不预填演示标题、描述、标签、分类或难度，所有内容应由用户输入。填写任意有效配置后触发自动保存；离开新建任务页前应立即保存未落库变更，补齐模板和数据集后自动保存的草稿可恢复完整配置。

字段：

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| 任务标题 | Input | 必填，2-80 字。 |
| 描述 | TextArea | 必填，面向任务广场和内部管理。 |
| 富文本说明 | Rich Text / Markdown 预览 | 面向标注员作答前阅读。 |
| 标签 | Input + Button + closable Tag list | 必填，每次输入一个标签并点击 `添加`；已添加标签在下一行以 Ant Design `Tag` 展示，可点击叉号移除。 |
| 分类 | Select multiple | 必填，可多选文本、图片、音频、视频；前端不再展示“多模态”选项，多个模态由所选标签共同表达。 |
| 难度 | Segmented / Select | 必填，简单、中等、困难。 |
| 截止日期 | DatePicker + Checkbox | 必填；可选择具体日期，也可勾选长期有效。 |
| 领取后完成时限 | InputNumber | 可不设置；设置后表示 Labeler 领取后需在对应小时数内完成。 |

发布后仅允许在已暂停状态修改 `description`、`rich_content`、`tags`。收集中、待审核和已结束任务仍必须允许进入详情页只读查看，其他字段展示为只读并给出原因提示，不得用“请先暂停发放”弹窗阻断查看。

### 6.4 模板与数据 Tab

目的：绑定已发布模板版本、选择数据集、完成 `ShowItem` 列映射。

区域：

- 模板选择：显示模板名称、版本、物料数量、是否已发布。
- 数据集选择：显示数据集名称、行数、字段数、可映射字段数。
- 列映射表：左侧为模板 `ShowItem`，右侧选择数据集列或渲染变量。
- 渲染预览：点击 `预览题目` 打开弹窗，使用同一 Renderer 展示样例题目。

模板版本和数据集两个选择框必须固定在模板与数据步骤内容区顶部。用户滚动查看模板详情、数据集详情或 ShowItem 映射表时，这两个选择框保持可见，避免选择状态丢失；固定选择区及 Select 输入框使用当前浅蓝模块渐变背景，不额外套白色框或阴影。

规则：

- 发布任务必须绑定已发布模板版本。
- 发布任务必须有题目或可由数据集生成题目。
- `ShowItem` 映射允许留空保存草稿，但发布前检查应阻塞。
- 修改已发布任务时模板、数据集和列映射只读。

### 6.5 题目管理 Tab

目的：查看任务题目、导入失败、题目状态和分配情况。

功能：

- 题目列表：题目 ID、序号、状态、领取人、提交状态、最近更新时间。
- 导入题目：JSON / JSONL / Excel；导入失败展示行号和错误原因。
- 批量编辑：仅草稿可批量删除、调整顺序或重导。
- 题目预览：打开抽屉查看原始 content 与 Renderer 预览。
- 题目导出：草稿阶段导出题目源数据；正式结果导出走导出中心。

状态提示：

- 草稿任务题目数为 0 时，发布按钮禁用。
- 已发布任务不允许删除题目，避免破坏领取和提交关系。

### 6.6 分发与奖励 Tab

字段：

- 配额：总题量、每人领取上限、每包题量。
- 奖励规则：`item` 按条或 `task` 按任务折算。输入框里的积分均表示标注员实际获得积分，不是企业最终支付成本。
- 分发策略：
  - `first_come_all`：包大小分配，与 Labeler 任务广场的领取包大小配置对齐；分享链接作为该策略下的可选开关展示，使用 `Switch` 开启后再填写有效期。有效期默认 72 小时，空值按 72 参与步骤校验；发布成功后的二维码、链接文本和“预览分享链接”按钮都展示完整同源 URL。
  - `quota_grab`：企业内流转，用于分配给企业内 Labeler；选择后展示 `Select mode="multiple"` 指定企业内 active Labeler，候选只能来自当前企业 active、非系统成员的 Labeler，留空表示所有企业 Labeler 可参与。
  - `assigned_link`：仅作为历史兼容策略，不在新建/修改任务的分发策略控件中作为独立选项展示。
- Labeler 分配比例：当企业内流转选择多位 Labeler 时，在选择框下方展示每位 Labeler 的任务分配百分比 `InputNumber`，复用人工复审多 Reviewer 百分比分配样式；合计必须为 `100%`，并按绑定数据集条数展示预计条数。选择 1 位 Labeler 时默认 `100%`，不选择 Labeler 时不展示比例，表示所有企业 active Labeler 可见可领。
- 资质规则：仅适用于 `first_come_all` 包大小分配，字段包含领域认证、最少完成任务数、历史通过率和说明。领域认证使用多选，当前固定选项为司法、心理、医疗、教育；不选择时显示无要求。`quota_grab` 企业内流转时必须隐藏所需资质领域、最低完成任务数、最低准确率和资质说明，保存 payload 时按空资质和 0 门槛处理。
- 任务可见范围：包大小分配进入个人公开任务广场；企业内流转进入企业项目工作台；分享链接仅作为包大小分配的辅助入口。

积分费用展示：

- 当分发策略为 `quota_grab` 企业内流转时，分发与奖励步骤必须隐藏奖励规则输入、平台手续费和费用估算，仅展示“企业内流转不分配积分”的 Ant Design `Alert`；发布 payload 使用 0 积分口径。
- 平台手续费率先写为前端常量 `10%`，标注员实际获得比例为 `90%`。
- 企业实际支付积分费用 = 标注员实际获得积分 / `0.9`；平台手续费 = 企业实际支付积分费用 - 标注员实际获得积分。
- 按条时，发布者填写标注员每通过 1 条数据实际获得的积分；页面在输入框下方展示标注员每条获得、企业每条预计支付、平台每条手续费和手续费率。
- 按任务时，发布者填写任务总奖励积分，系统优先用绑定数据集 `row_count` 作为标准条数折算每条奖励；实际仍按通过条数结算，不表示已实现多人复杂分配、权重分配或人工结算。未绑定数据集或 `row_count=0` 时显示 `选择数据集后计算标准条数` 并按发布阻塞处理。
- 右侧发布摘要、确认发布页和发布前检查 Modal 必须展示同一套费用信息，并明确区分标注员获得积分、平台手续费和企业实际支付积分费用。所有数值最多保留 2 位小数；未填写、为 0 或非法时显示 `未填写` / `待填写` / `待计算`，不能显示 `NaN`。
- 积分费用估算框保持紧凑展示，不在标题右侧重复展示手续费 Tag，也不在底部追加解释性说明；手续费率和费用含义应通过明细项表达。费用数值单位统一显示为 `积分`，不要使用 `分`。

发布后这些字段只读；如必须调整，应通过新建任务副本或后续版本机制处理。

任务管理列表的行级 `更多` 菜单中，`quota_grab` 任务必须提供 `分配企业 Labeler` 动作。点击后打开 Ant Design `Modal`，展示当前分配范围、当前分配比例和企业 Labeler 多选框；选择多位 Labeler 时展示同样的百分比分配列表，提交后调用 `PUT /tasks/{task_id}/internal-labelers` 并写入 `assignment.target_labeler_ids` / `assignment.target_labeler_allocations`。已结束任务该动作置灰；非企业内流转任务不展示该动作。

### 6.7 AI 预审 Tab

- 是否启用 AI 预审。
- 选择企业已配置的 AI Provider；Provider 已封装 Base URL、API Key、Temperature 和对应模型，任务发布页只展示 Provider 派生模型，不再单独选择审核模型。
- 如果任务类型为图片、音频、视频或多模态，而所选 Provider 的模型名称/备注未体现对应模态能力，页面必须用 Ant Design `Alert` 显示警告，提醒发布者到资源配置确认能力。
- 维度选择使用 Ant Design `Checkbox.Group`，预设维度包括准确性、完整性、一致性、格式规范、证据充分性、逻辑合理性、安全合规；自定义维度用 `Input + Button` 添加，并用 `Tag` 展示和删除。
- Input 字段说明由 AI 根据数据集名称、模板名称、字段样例、ShowItem 映射和模板答案字段上下文生成，避免只把不易识别的变量名直出给大模型；页面保留 `AI 生成字段说明` / `AI 重新生成` 按钮、`Button loading` 状态和可编辑 `TextArea`。
- 审核评分矩阵使用 Ant Design `Table` 展示和编辑，每个维度必须显示定义、评分标准、扣分规则、打回条件、人工复核条件；`AI 生成矩阵`、`AI 重新生成` 和确认矩阵使用 `Button loading`/主按钮状态，当前按钮作为 AI Gateway 生成接口预留。
- 自动判定阈值拆为通过阈值、打回阈值、人工复核下限和人工复核上限，使用 `InputNumber` 并显示 `%`。
- Output 要求完全不在发布页面展示，不提供说明卡、Tag、Prompt 或 JSON schema；输出契约由后台 Prompt Engine 和 function call 配置维护。
- 失败策略：重试次数、失败后进入人工复核。
- 成本预估：调用 `AI Resources` 估算 token 和预算。

规则：

- 业务页面不直接调用第三方模型；所有 AI 调用必须走 AI Resources / AI Gateway。
- AI 输出需要结构化结果，至少包含维度评分、总分、建议和结论。
- 发布前 AI 配置若启用但缺少 Provider、字段说明、评分矩阵、矩阵确认或有效阈值，应阻塞发布；不再要求发布者单独选择审核模型。

### 6.8 人工复审 Tab

- 审核员选择：使用 Ant Design `Select mode="multiple"` 选择企业内 active Reviewer，允许为空后续分配，不再使用手填 ID/邮箱文本框。
- 审核员分配比例：当选择多位 Reviewer 时，在选择框下方展示每位 Reviewer 的百分比 `InputNumber`，合计必须为 `100%`，并按绑定数据集条数展示预计条数。前端提交时继续写 `reviewer_ids`，每人百分比写入 `review_config.reviewer_allocations`。
- 审核阶段：初审、复审、终审预留。
- 审核分配：平均分配、按领域、手动指派预留。
- 打回规则：是否允许 Labeler 修改后重提、最大轮次。

资质领取前检查属于“分发策略”步骤，不再放在单独审核步骤中。

### 6.9 用户协议 Tab

- 默认开启“领取前协议”，可关闭。
- 支持勾选默认任务用户协议模板。
- 支持 `Input.TextArea` 编辑自定义协议正文。
- 支持 `Upload` 选择协议文件；TXT、Markdown、CSV、JSON、HTML 等文本类文件必须解析并回填到协议正文输入框，同时关闭默认协议模板。PDF、Word 等二进制文档在未接入后端解析服务前不允许作为正文占位上传，需提示用户先转换为文本后再上传。
- 任务广场详情抽屉领取流程必须拆为 `任务详情 -> 签署协议 -> 领取确认` 三步；第二步显示“我已阅读并同意该任务用户协议”勾选框，未勾选必签协议时不能进入领取确认，后端也必须拒绝未同意领取。

### 6.10 统计与导出 Tab

展示任务运行期数据：

- 领取数、提交数、通过数、打回数、待审核数。
- AI 预审任务状态：pending、processing、completed、failed。
- 质量趋势：通过率、打回率、人工复核率。
- 配额消耗和截止风险。

导出：

- 创建异步导出任务：JSON、JSONL、CSV、Excel。
- 字段配置：include/exclude、重命名、是否包含审核记录、是否包含 AI 结果。
- 过滤：通过、打回、待审、时间范围、标注员、Reviewer。
- 下载历史：导出状态、进度、创建人、文件大小、下载入口。

导出不应直接下载同步文件，应调用 `/exports` 创建异步任务。

### 6.11 操作日志 Tab

展示关键审计事件：

- 创建任务、保存草稿、发布、暂停、恢复、结束。
- 模板/数据集绑定变化。
- 审核员、AI、资质、分发配置变化。
- 导出创建和下载。
- 删除草稿或复制任务。

每条日志包含操作人、时间、动作、变更摘要和 request_id。

## 7. 发布前检查抽屉

点击 `发布任务` 后打开右侧 Drawer，而不是立即发布。

检查项：

| 检查项 | 阻塞规则 |
| --- | --- |
| 基础信息 | 标题、描述、截止时间、配额缺失阻塞。 |
| 模板 | 未选择已发布模板阻塞。 |
| 数据集/题目 | 无题目或数据集解析失败阻塞。 |
| 列映射 | 必需 `ShowItem` 未映射阻塞。 |
| 审核员 | 开启人工审核但无 Reviewer 阻塞。 |
| AI 预审 | 启用 AI 但缺 Prompt、维度或预算不足阻塞。 |
| 资质 | 规则冲突或无可领取人时警告。 |
| 分发 | 包大小分配开启分享但缺有效期、企业内流转指定了无效 Labeler、多 Labeler 分配比例未合计 100%、或配额配置错误阻塞。 |

抽屉底部：

- 阻塞项存在：主按钮禁用，显示 `完成检查项后发布`。
- 仅警告项：允许发布，但需要二次确认。
- 全部通过：主按钮 `确认发布`，调用 `POST /tasks/{task_id}/publish`。

发布成功后：

- 状态变为 `published`。
- 新建/草稿发布向导自动返回任务管理列表，顶部显示发布成功提示，并刷新列表展示最新任务状态。
- 写入操作日志。

## 8. 删除、结束与复制

### 8.1 删除草稿

- 仅草稿任务可删除。
- 删除前弹窗显示任务名、题目数、关联模板和数据集。
- 删除草稿的二次确认框必须页面居中显示。
- 用户需确认影响：删除后不可恢复，但不删除原始数据集和模板。

### 8.2 结束任务

- 发布中或暂停任务可结束。
- 结束后不可恢复为发布中。
- 弹窗提示是否立即创建导出任务。
- 结束动作写入审计日志。

### 8.3 复制任务

- 复制生成新的草稿任务。
- 默认复制基础信息、模板版本、数据集、列映射、审核和 AI 配置。
- 不复制提交、领取、审核记录和导出历史。
- 新任务标题默认追加 `副本`，进入修改子页面继续编辑。

当前已落地：列表行级 `更多 / 复制任务` 调用 `POST /tasks/{task_id}/copy`，成功后展示提示并直接打开新草稿修改子页面。后端会复制源任务题目内容快照，使副本可在不重新导入数据集的情况下继续编辑、发布检查和题目预览；副本的统计从 0 开始，领取、提交、审核记录和导出历史不被复制。

## 9. 空态、加载与错误

### 9.1 列表空态

无任务时显示：

- 标题：`还没有任务`
- 说明：`新建任务草稿，配置模板与数据、分发奖励和用户协议后即可进入发布流程。`
- 主按钮：`新建任务`
- 次按钮：`查看模板搭建`、`查看数据集管理`

### 9.2 子页面错误

- 任务不存在：显示 `任务不存在或已被删除`，提供返回列表按钮。
- 权限不足：显示 `没有该任务的管理权限`，提示联系企业管理员。
- 状态冲突：例如已发布任务尝试修改模板，字段旁展示只读原因。
- 保存失败：错误展示在对应 Tab 和字段附近，顶部仅做汇总。

### 9.3 加载态

- 列表加载：Table skeleton。
- 子页面加载：标题骨架 + Tabs 内容骨架。
- 保存/发布：按钮 loading，禁止重复提交。

## 10. 响应式与可访问性

- 桌面端优先覆盖 1280、1440、1920 宽度。
- 任务表格支持横向滚动，固定任务名称列和操作列。
- 窄屏下筛选进入 Drawer，表格操作收进 `更多`。
- 所有 icon-only 操作必须有 tooltip 或 `aria-label`。
- 危险操作使用 `danger`，并通过文字明确影响，不能只靠红色。
- 键盘用户应能从列表进入修改、保存草稿、打开发布检查和关闭弹窗。

## 12. Hallmark 反模板化检查

本页应避免以下问题：

- 不把任务管理做成“顶部几个彩色卡片 + 卡片列表”的通用后台模板。
- 不在列表页塞完整创建表单；创建/修改必须进入子页面。
- 不把发布按钮做成无校验的普通提交；必须有发布前检查抽屉。
- 不把导出做成同步下载按钮；必须体现异步导出和下载历史。
- 不伪造通过率、质量分、AI 成本等指标；接口无数据时显示 `-` 或明确占位。
- 不绕过状态机：已发布任务的敏感字段只读，删除只限草稿。

## 13. 实施优先级

第一阶段：

- 任务管理列表页、筛选、状态标签、行操作入口。
- 新建/修改子页面框架，基础信息、模板与数据、发布配置、审核与 AI、资质与分发 Tabs。
- 发布前检查抽屉和状态机操作按钮。

第二阶段：

- 题目管理 Tab、导入失败行号展示、Renderer 样例预览。
- 统计与导出 Tab、导出配置弹窗和导出历史。
- 操作日志 Tab。

第三阶段：

- 深链接路由、批量操作、复制任务、领取资格模拟、AI 成本预估和 WebSocket 实时统计。
