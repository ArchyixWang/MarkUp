# 公开帮助手册设计稿

## 页面定位

`/help` 是 MarkUp 面向终端用户的公开帮助手册，不是营销落地页、开发者文档、API 文档或部署文档。页面内容继续来自 `apps/web/src/pages/help/helpContent.json`，该 JSON 同时作为平台问答 AI 的公开知识源。正文事实必须以当前已开发功能、活跃产品/API/架构文档和 `docs/MarkUp-说明文档/` 为依据；布局参考 Gitea Docs 只是阅读结构参考，不能只改排版而保留过期或泛化文字。

本轮参考 `https://docs.gitea.com/` 的文档站结构和正文格式风格，吸收其左侧文档目录、正文文章、右侧本页目录、短段落、列表和 warning/callout 心智，但配色保持 MarkUp 蓝色主色调，不复制 Gitea 绿色品牌。

正文文字风格参考 Gitea 中文文档的手册写法：每页只解释一个主题，开头用短段说明用途，后续用清晰小节承载“使用前准备 / 操作步骤 / 注意事项 / 常见情况”，语句直接描述如何操作和为什么受限，不写营销式价值表达。

2026-06-09 的正文基线已经按当前交付说明重写：覆盖个人/企业 Labeler 分流、企业内流转与分配比例、Owner 发布进入待审核、Team Admin 审批、Manifest JSONL 与多模态上下文、模板 Designer/Renderer/版本、AI worker 闭环、人工审核、多轮审核记录、同步生成但按异步模型展示的导出、企业会员/积分钱包/AI 钱包和异常排查。后续新增功能或变更状态机时，必须同步更新 `helpContent.json` 和本文。

2026-06-09 后续结构重构：`/help` 从 9 个大章节改为模块优先帮助中心，内容模型由 `sections/cards` 切换为 `groups/modules/entries/faqs/aliases`。公开范围继续只覆盖 Labeler、Owner、Reviewer、Team Admin，不纳入内部运营后台操作。模块目录按功能域组织为公开入口与账号体系、身份角色权限、任务广场、标注作答工作台、资质积分信誉、企业工作台与 Dashboard、数据集、模板、任务发布、任务管理、AI、人工审核、导出文件审计、企业治理成员钱包资源、通知公告个人信箱、常见失败排查，以及独立 FAQ；正文必须用连续段落讲清功能在平台链路里的位置、入口、流程、限制和常见失败原因，不能把所有说明拆成碎片化 bullet 清单。

2026-06-10 操作手册修订：公开手册正文从“功能介绍”进一步改为“可照着做的使用指引”。账号部分明确用户身份唯一，入口由当前账号的成员状态和角色授权决定，不描述多企业或企业/个人身份切换。数据集部分必须分别讲清 CSV/Excel 普通文本导入、表格媒体 URL、Manifest JSONL、多模态媒体与表格行关联、补上传按主值合并、未绑定素材、`media_schema`、`context_schema` 与 ShowItem 映射关系。模板和标注作答部分必须覆盖真实组件：ShowItem、TextInput、TextArea、SingleSelect、MultiSelect、TagSelect、Scale、Ranking、RichEditor、FileUpload、ImageUpload、AudioUpload、VideoUpload、JsonEditor、ImageMaskAnnotation、LLMComponent、GroupContainer，尤其要把图片 Mask 的底图来源、绘制、修正和提交前检查写成操作步骤。

## 信息架构

1. 文档头部：显示 `MarkUp 帮助手册`、最后更新时间和全文搜索；不展示额外受众说明、布局说明或知识源说明。
2. 常用入口：以轻量入口卡展示任务领取、数据集管理、模板搭建、任务发布、AI 与审核、企业资源；入口卡只保留标题、图标和目标模块标识，不放解释性长描述。
3. 角色筛选：移动端在正文上方使用横向 Ant Design `Segmented`，桌面左侧栏不再展示单独的角色筛选块，避免挤占文档目录阅读空间。
4. 模块目录：桌面左侧目录按 `groups` 分组展示当前筛选/搜索后的模块列表，模块项显示条目数量或 FAQ 数量；搜索命中后目录只保留命中的模块。
5. 搜索结果：搜索后正文上方显示命中的模块、条目和 FAQ 摘要列表，点击结果跳转对应模块；搜索状态不再只依赖左侧目录变化。
6. 正文文章区：每次只展示一个模块或 FAQ 页；搜索命中后当前页仍展示完整正文，不裁剪为局部结果，并在标题、摘要、段落、列表项、FAQ 问答和标签中高亮命中的关键词。
7. 本页目录：宽桌面右侧展示当前页内部小节目录，不额外放阅读建议提示框。
8. 页底翻页：正文底部只提供上一页和下一页两个文档跳转按钮，不展示数字页码条；上一页/下一页标题必须完整显示，不能被省略或挤成多行。
9. FAQ：FAQ 作为独立模块页，保留 Ant Design `Collapse`；搜索命中任意 FAQ 时目录显示 FAQ 页，进入后展示当前角色可见的完整 FAQ 并高亮关键词。

## 内容格式

- 正文优先使用短段落和列表，不把长说明压缩成同质卡片墙。
- 有顺序的流程使用编号步骤，例如任务领取、任务发布、审核动作。
- 无顺序的知识点使用图标 bullet，突出可扫读性。
- 章节文案统一使用说明文档语气：先说明适用场景，再写操作步骤、约束和排查项；避免“能力介绍式”长句和宣传语。
- 模块必须分页呈现，禁止把全部手册模块和 FAQ 一次性堆在同一个长滚动页面里。
- 公开手册只描述当前已经完成并面向终端用户开放的能力；不要把已完成能力写成 `建设中`、`planned` 或后续增强方向。
- 内部运营、API、部署、本地启动和开发者说明仍留在活跃 `docs/`，不得进入公开 `/help`。

## 视觉与组件

- 主色：继续使用平台 `#2563eb` 蓝色；白底、浅蓝头部、浅边框和 8px 圆角。
- 组件：Ant Design `Input.Search`、`Segmented`、`Anchor`、`Tag`、`Alert`、`Collapse`、`Empty`、`Button`、`Typography`，图标使用 `@ant-design/icons`。
- 正文行宽控制在约 78ch，便于连续阅读。
- 常用入口卡只作为导航入口，不承担营销卖点；卡片采用浅边框、短标题、短说明和入口元信息。
- 桌面端优先使用三栏文档布局；中等宽度降为左目录 + 正文；移动端降为单列。

## 响应式与可访问性

- 390px 起不得出现页面级横向滚动。
- 移动端隐藏右侧本页目录和左侧文档分页目录，只保留顶部角色筛选、当前页正文和页底翻页。
- 常用入口是原生 `button`，支持键盘焦点；点击后更新 hash 并切换到对应文档页。
- 正文顶部不展示“全部文档 / N 个文档页”等搜索结果提示条，搜索状态只通过搜索框内容、左侧目录、当前正文变化和关键词高亮体现。
- FAQ、目录和搜索继续使用 Ant Design 原生键盘与可访问语义。
