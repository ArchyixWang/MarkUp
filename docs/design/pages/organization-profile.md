# 企业信息页面设计稿

## 1. 页面定位

企业信息页是企业面板下的资料维护子页面，用于维护当前企业/企业的企业资料，不承担企业总览、治理入口、跨模块跳转或生产判断职责。

页面只做 4 块内容：

- 基本信息
- 开票信息
- 邮寄信息
- 企业认证

其中基本信息、开票信息、邮寄信息是主体，企业认证作为第四块独立维护模块保留。企业会员只在基本信息内展示只读 Tag 和到期时间，不在本页提供购买、续费或降级入口。

## 2. 入口与面包屑

侧栏：

- 分组：`企业管理`
- 条目：`企业信息`

面包屑：

```text
工作台 / 企业信息
```

## 3. 页面结构

```text
企业信息
├─ 固定页面标题
├─ 页面操作：刷新 / 保存修改
├─ 基本信息卡片
├─ 开票信息卡片
├─ 邮寄信息卡片
└─ 企业认证卡片
```

页面主体位于固定工作台视口内的内部滚动区，不使用 Hero、状态总览、治理快照或底部治理入口。

桌面端：

- 首屏第一行固定为 `基本信息 + 企业认证` 并列，保证认证状态、审核意见和材料入口默认可见。
- `基本信息`、`开票信息`、`邮寄信息` 在默认查看态使用“摘要头部 + 静态资料行”的轻量资料卡，不直接平铺成大段表单或纯 `Descriptions` 清单。
- 点击 `编辑资料` 后，上述三块才切换为 `Form + Row/Col` 的可编辑表单。
- 字段顺序稳定，便于快速维护。

窄宽度下：

- 顺序回落为 `基本信息 -> 企业认证 -> 开票信息 -> 邮寄信息`。
- 不做复杂双栏编排。

## 4. 页面头部

左侧：

- 标题：`企业信息`
- 副文案：`维护企业基本资料、开票信息、邮寄信息与认证信息。`

右侧：

- `刷新`
- `编辑资料`
- 编辑态下显示 `取消编辑` / `保存修改`

不展示：

- 就绪度
- 推荐动作
- 资源摘要
- 成员快照
- 日志快照

## 5. 基本信息

继续复用当前企业基础字段：

- 企业名称
- 行业
- 联系电话
- 官网
- 地址
- 会员套餐（只读）
- 企业简介
- Logo URL
- Logo 上传

规则：

- `企业名称` 必填。
- `官网` 使用 URL 校验提示。
- `会员套餐` 来源于企业详情 `membership`，只展示当前有效套餐、到期时间、已到期提示和预约降级提示；购买入口固定放在资源配置页 `会员与额度`。
- Logo 上传成功后回填 `logo_url`，仍需点击 `保存修改` 才写回企业资料。
- Logo 上传控件本地仅接受 JPG、PNG 或 GIF 图片；PDF、可执行文件等非图片文件必须在发起上传前拦截。
- 页面默认只读，点击 `编辑资料` 后才进入基础资料、开票信息和邮寄信息的可编辑状态。

## 6. 开票信息

新增 1 套固定开票资料，不做多套管理：

- 发票类型
- 发票抬头
- 税号
- 开票地址
- 开票电话
- 开票邮箱
- 开户行
- 银行账号
- 备注

交互原则：

- 只是资料维护，不做发票记录、历史版本或默认抬头切换。
- 首版使用普通文本表单，不引入复杂业务流。

## 7. 邮寄信息

新增 1 套固定邮寄资料，不做地址簿：

- 收件人
- 收件电话
- 所在地区
- 详细地址
- 邮编
- 地址别名
- 默认地址

说明：

- `所在地区` 首版为普通文本字段。
- 不接省市区级联数据源。

## 8. 企业认证

企业认证作为第四块独立模块保留，但降为资料维护的一部分，不再作为页面主叙事。

卡片展示：

- 认证状态
- 最近提交时间
- 企业主体名称
- 统一社会信用代码
- 认证联系人
- 联系电话
- 材料数量
- 审核意见

操作：

- `提交认证`
- `编辑认证信息`
- `补件并重新提交`
- `查看材料`

交互：

- 认证编辑继续使用 `Modal` 或 `Drawer`。
- 认证材料必须通过上传文件产生，用户只看到“上传材料文件”和已上传文件列表，不暴露可手填的 URL 文本域。
- 材料查看继续使用独立 `Drawer`，展示已提交材料文件名、访问地址和查看文件入口，不再提示“材料接口未接入”。
- 不改成页面常驻大表单。

## 9. 交互与反馈

- 页面级成功/失败反馈统一使用 Ant Design `message`。
- 错误态仍用页面内 `Alert`，但不插入大量成功提示影响布局。
- 保存资料后保持页面高度稳定，不因提示出现和关闭发生跳动。
- 页面默认查看态，只有显式点击 `编辑资料` 才允许修改基础资料、开票信息和邮寄信息。
- 默认查看态不使用灰色 disabled 表单冒充展示态；空值统一显示 `-`。
- 普通成员只读，仍展示同一份静态资料结构，不显示 `编辑资料`、`取消编辑`、`保存修改`。

## 10. Ant Design 组件建议

- `Card`：四个资料模块容器。
- `Form`：基本信息、开票信息、邮寄信息。
- `Row/Col`：字段两列布局。
- `Input` / `Input.TextArea`
- `Radio.Group`
- `Upload`
- `Descriptions`
- `Alert`
- `Modal`
- `Drawer`
- `message`

## 11. 验收标准

- 首屏不再出现就绪度、推荐动作、治理入口、预算快照。
- 页面稳定显示 4 个模块：基本信息、开票信息、邮寄信息、企业认证。
- 修改基本信息、开票信息、邮寄信息后可保存并刷新保留。
- 企业认证仍可提交、补件、查看材料。
- 成功/失败提示不改变页面整体宽高。
- 页面在桌面端和窄宽度下都保持单一资料维护心智，不再像子控制台首页。
