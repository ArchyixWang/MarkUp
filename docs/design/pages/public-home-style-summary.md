# MarkUp 首页视觉风格总结文档

## 1. 当前视觉基线

MarkUp 当前整体风格是“浅蓝 + 白底 + 生产工作台”路线，未登录页允许稍强一点的品牌表达，但不能偏离数据标注平台的工具属性。整体应该传达稳定、可信、可追溯，而不是营销页式的大开大合。

## 2. 色彩体系

- 主背景：白色、极浅蓝灰，保持通透。
- 主品牌色：蓝色，用于首页主 CTA、链接、AI 相关强调。
- 辅助强调：橙色只用于少量关键提示或品牌点缀，不能压过蓝色。
- 成功状态：绿色。
- 警告状态：琥珀或橙黄。
- 错误状态：红色。
- 分割与边框：浅灰线，不使用重阴影。

首页应延续现有工作台的干净浅色底色，同时允许首屏有更强的产品感和层次感，但不能出现大面积彩色渐变或夸张发光效果。

## 3. 字体与层级

- 仍以系统中文字体栈为主，和当前 Ant Design 全局主题保持一致。
- 大标题要短、稳、明确，不要营销口吻。
- 字号层级保持克制：Hero 标题大，正文和说明紧凑，避免过多花体或装饰性字重。
- 英文标签可以出现，但不能取代中文产品语义。

## 4. 按钮与交互风格

- 主按钮使用 Ant Design `Button type="primary"` 的语义。
- 次按钮使用浅底或描边按钮。
- 首页 CTA 需要明确区分“开始发布任务 / 浏览任务市场 / 进入工作台”。
- 交互状态应清晰：hover 轻抬升、focus 可见、disabled 明确。
- 不引入拟物化按钮、发光按钮或夸张渐变按钮。

## 5. 卡片、表格、面板风格

- 首页卡片应延续工作台风格：8px 圆角、浅边框、白底、轻阴影。
- 不做厚重卡片墙，不堆叠很多浮卡。
- 适合用 Ant Design `Card`、`Statistic`、`Tag`、`Progress`、`Steps` 组织信息。
- 首页产品预览应像工作台中的真实信息面板，而不是插画式海报。

## 6. 图标与插画策略

- 图标优先使用 `@ant-design/icons`，少量关键位置可使用平台内部图标。
- 首页主视觉不依赖插画，改为产品界面式的组合预览。
- 如果需要强调平台能力，用组件化界面拼块而非纯装饰图形。

## 7. 留白、圆角、阴影、边框

- 留白要明确，但不能松散。
- 圆角统一在 8px 左右。
- 阴影轻、短、克制，只用于层次分离。
- 边框以浅灰和淡蓝为主，不使用厚边框。

## 8. Ant Design 使用习惯

- 首页尽量沿用 Ant Design 基础组件，不自造整套 UI。
- 把产品信息装入 `Card` / `Descriptions` / `Statistic` / `Steps` / `Tag` / `Progress`。
- 按钮、标签、提示、状态条都遵循现有主题色。
- 首页可以比工作台更有品牌感，但仍应明显属于同一系统。

## 9. 已有 AI 助手和工作台的视觉特征

- 模版搭建 AI 与任务发布 AI 已采用固定悬浮入口、对话面板、变更确认和预览模式。
- 工作台页面强调固定标题栏、信息密度、内部滚动和清晰的状态条。
- 首页应复用这套“工具型、可信赖、清晰状态”的审美，而不是重新走营销首页路线。

## 10. 结论

首页视觉关键词应是：浅蓝、白底、工作台感、生产链路、稳定、可信、少装饰、强结构。
