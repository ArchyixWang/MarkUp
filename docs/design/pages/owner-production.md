## 信息架构

Owner 生产页优先服务高频生产动作，不做营销页式展示：

- 数据集管理：导入数据、字段账本、渲染变量、预览、下载。
- 模板搭建：物料区、画布、属性面板、Renderer 预览、已保存模板列表。
- 发布任务：模板选择、数据集映射、分发与资质、积分、AI 预审、人工复审、发布摘要。

## Ant Design 使用原则

- 表格：默认使用 `Table` 管理数据集、模板列表和任务列表，固定表头、内部滚动和右下角分页。
- 卡片：任务管理、模板搭建、数据集管理可通过筛选栏右侧 `Segmented` 切换到 `Card` 网格，卡片样式参考任务广场但使用企业端紧凑信息密度。
- 表单：使用 `Form`、`Input`、`Select`、`Checkbox`、`Upload`。
- 弹窗/抽屉：短表单使用 `Modal`，详情和发布前复核可使用 `Drawer`。
- 状态：使用 `Tag`、`Alert`、`Progress` 和 `Statistic`。
- 操作：主动作使用 `Button type="primary"`，危险操作使用 `danger`。

## 交互约束

- 数据集导入和新增渲染变量应由按钮触发弹窗，不在主页面长期展开。
- 任务、模板、数据集列表页的表格/卡片切换必须复用同一搜索筛选条件；卡片区内部滚动，底部固定 `Pagination`，并启用每页条数与快速跳转。
- 模板 Renderer 预览使用独立弹窗或抽屉，不挤占右侧属性配置面板。
- 模板删除全部组件后必须显示空态和恢复入口，不能白屏。
- 模板 Designer 中单选、多选和标签选择的选项编辑使用 Ant Design 多行输入，支持回车逐行录入选项；保存 schema 时过滤空行。
- 模板列表中的已发布模板不再在前端暴露“归档”入口，只保留删除；删除仍由后端判定是否已有任务引用。已发布模板点击“新建版本/修改”后，Designer 必须基于当前已发布 schema 生成新的草稿版本，而不是回退到空白模板或陈旧 schema。
- 发布任务页必须提供发布前摘要，覆盖配额、截止时间、分发策略、奖励、模板、映射、审核员、AI 阈值和资质要求。
